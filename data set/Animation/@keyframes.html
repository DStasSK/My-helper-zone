Ключевые кадры анимации @keyframes

Правило @keyframes устанавливает ключевые кадры при анимации элемента. Это правило включает имя анимации, любое число контрольных точек, а также свойства, предназначенные для анимации. Контрольная точка это набор css свойств элемента (прозрачность, цвет, положение и др.), которые должны применяться к элементу в заданный момент времени. Вычисление промежуточных значений между кадрами производит браузер.

@keyframes step {
  from { left: 0; }       /* соответстует 0% времени - началу анимации */
  to   { left: 100px; }   /* соответствует 100% времени - окончание анимации */
}
@keyframes slide {
    0% { left:0;    top:0; }     /* начало анимации */
   50% { left:50%;  top:50%; }   /* промежуточное состояние свойств элемента */
  100% { left:100%; top:0; }     /* окончание анимации */
}

Контрольные точки ключевых кадров задаются помощью процентов, начиная с 0% и работая до 100%, с промежуточными точками между 0% и 100%. При желании вместо 0% и 100% могут быть использованы ключевые слова from и to, а промежуточные точки указывать в процентах.

"Правила" обработки ключевых кадров:
  • Ключевые кадры не обязательно должны начинаться с 0% и заканчиваться 100%. Анимация тогда будет происходить не сразу. При этом общая длительность анимации будет попрежнему расчитана от 0% до 100%, а браузеры будут использовать существующие стили элемента для начального/конечного состояния.
  • Если ключевой кадр определен несколько раз (например 50%), но не все затронутые свойства присутствуют в каждом ключевом кадре, то будут учитываться все значения, указанные в этих ключевых кадрах.
  • Если в ключевом кадре указать свойства, которые не могут быть анимированы, то они будут игнорируются.
  • Если в ключевом кадре css свойство указано совмесно с параметром !important, то оно игнорируется.
  • Если название анимации в css файле встречается несколько раз, то будет исспользоваться последняя, ключевый кадры из предыдущих блоков учитываться не будут.
Для каждогй ключевой точки анимации допускается указывать свой тип воспроизведения (определяемый параметром animation-timing-function), при этом указанный тип анимации будет воспроизводиться в течении данного ключевого кадра, переопределяя глобальный параметр, заданный у селектора.

Важно отметить, что могут быть анимированы только отдельные свойства, указанные к первом ключевом кадре. Например: попытка анимировать от top: 0 до bottom: 0 не будет работать, потому что анимация может применять только переход в пределах одного свойства, а не от одного свойства к другому. В этом случае элемент необходимо анимировать от top: 0 до top: 100%.

Правило @keyframes может встречаться с вендорными префиксами:
  • @-webkit-keyframes
  • @-moz-keyframes
  • @-o-keyframes
