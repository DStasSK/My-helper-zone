CSS cвойство touch-action

Свойство touch-action устанавливает, как пользователь сенсорного экрана может манипулировать областью элемента (например, с помощью встроенных в браузер функций масштабирования).

Приложения, использующие события Touch events, отключают обработку браузером жестов, вызывая preventDefault(), также должны использовать touch-action, чтобы браузер знал намерение приложения до того, как будут вызваны какие-либо прослушиватели событий.

После начала жеста изменения touch-action не окажут никакого влияния на поведение текущего жеста.
div {
  touch-action: auto | none | pan-x | pan-left | pan-right | pan-y |
                pan-up | pan-down | pinch-zoom | manipulation;
	}
  • auto - включить обработку браузером всех жестов панорамирования и масштабирования.
  • none - отключить обработку браузером всех жестов панорамирования и масштабирования. Может препятствовать работе возможностей масштабирования браузера. Это не позволит людям с плохим зрением читать и понимать содержимое страницы.
  • pan-x - включить жесты горизонтального панорамирования одним пальцем. Можно сочетать с pan-y, pan-up, pan-down и/или pinch-zoom.
  • pan-y - включить жесты вертикального панорамирования одним пальцем. Можно сочетать с pan-x, pan-left, pan-right и/или pinch-zoom.
  • manipulation - включить жесты панорамирования и масштабирования, но отключить дополнительные нестандартные жесты, такие как двойное нажатие для масштабирования. Отключение двойного нажатия для масштабирования устраняет необходимость для браузеров задерживать генерацию событий щелчка, когда пользователь касается экрана. Это псевдоним для "pan-x pan-y pinch-zoom".
  • pan-left, pan-right, pan-up, pan-down - включить жесты одним пальцем, которые начинаются с прокрутки в заданном направлении(ях). После начала прокрутки направление все еще может быть обратным. Обратите внимание, что прокрутка «вверх» (pan-up) означает, что пользователь проводит пальцем вниз по поверхности экрана, а pan-left означает, что пользователь проводит пальцем вправо. Несколько направлений могут быть объединены, за исключением случаев, когда есть более простое представление (например, «pan-left pan-right» недопустимо, поскольку «pan-x» проще, но «pan-left pan-down» допустимо).
    данные значения не поддерживаются некоторыми браузерами.
  • pinch-zoom - включить панорамирование и масштабирование страницы несколькими пальцами. Это можно объединить с любым из значений pan-*.

По умолчанию жесты панорамирования (прокрутки) и сжатия обрабатываются исключительно браузером. Приложение, использующее события Pointer events, получит событие pointercancel, когда браузер начнет обрабатывать жест касания. Явно указав, какие жесты должны обрабатываться браузером, приложение может предоставить собственное поведение в прослушивателях pointermove и pointerup для оставшихся жестов.

При запуске жеста браузер пересекает значения touch-action затронутого элемента и его предков, вплоть до того, который реализует жест (другими словами, первого содержащего прокручиваемый элемент). Это означает, что на практике touch-action обычно применяется только к элементам верхнего уровня, которые имеют какое-либо пользовательское поведение, без необходимости явно указывать touch-action для любого из потомков этого элемента.
