@charset

Правило CSS @charset определяет кодировку символов, используемую в таблице стилей. Этот синтаксис полезен при использовании не-ASCII-символов в некоторых свойствах CSS, таких как content и при использовании символов национального алфавита. Хотя первым символом в @charset является символ @, это не правило at.

Правило CSS @charset - это определенная последовательность байтов, которая может быть размещена только в самом начале таблицы стилей. Никакие другие символы, кроме метки порядка байтов Unicode, не допускаются перед ней. Он также не следует обычным правилам синтаксиса CSS, таким как использование кавычек или пробелов.
Для внешней таблицы стилей браузер последовательно просматривает следующие пункты для определения кодировки таблицы стилей, чем выше находится пункт, тем выше его приоритет:
  • кодировка, которую отдает сервер.
  • правило @charset из начала css файла.
  • атрибут charset элемента link (устарелo, не используйте его).
  • если ни один из пунктов не найден - установит UTF-8.
В css файле для русского языка обычно указывается кодировка windows-1251 или utf-8. Значение кодировки обязательно должно быть взято в кавычки.
@charset "windows-1251";

body {
  font: 11pt Arial, Helvetica, sans-serif;
  margin: 0;
}

/* ... */
@charset 'iso-8859-15';     /* ошибка - нужны двойные кавычки */
@charset  "UTF-8";          /* ошибка - лишний пробел перед значением */
 @charset "UTF-8";          /* ошибка - лишний пробел перед правилом */
@charset UTF-8;             /* ошибка - нет двойных кавычек */
