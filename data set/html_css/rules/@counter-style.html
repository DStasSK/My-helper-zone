@counter-style

Правило @counter-style позволяет расширять предопределенные стили списков и определять собственные стили счетчиков ul и ol, когда предопределенные стили не соответствуют их требованиям.

@counter-style <name> {
  <declaration-list>
}
  • <name> - идентифицирует указанные стили - позволяет к ним в последующем обратиться. Имя группы свойств указывается без кавычек; и оно чуствительно к регистру. Оно не может быть none или совпадать с названием CSS свойств и их ключевыми словами.
  • <declaration-list> - перечень определенных CSS свойств и их параметров.
    • system - указывает алгоритм преобразования значения счетчика в строковое представление. Если значение задано как cyclic, numeric, alphabetic, symbolic или fixed необходимо будет также указать дескриптор символов - symbols. Если значение задано как additive, необходимо также указать дескриптор аддитивных символов - additive-symbols.
    • symbols - указывает символы, которые будут использоваться для представления маркеров. Символы могут содержать строки, изображения или пользовательские идентификаторы. Этот дескриптор требуется, если свойство system имеет значение cyclic, numeric, alphabetic, symbolic или fixed. 
      symbols: A B C D E;
      symbols: "\24B6" "\24B7" "\24B8" D E;
      symbols: "0" "1" "2" "4" "5" "6" "7" "8" "9";
      symbols: url("one.svg") url("two.svg") url("three.svg");
    • additive-symbols - в качестве значения указывается список символов счетчика, перечисленнымх в порядке убывания. Этот дескриптор требуется, если свойство system имеет значение additive-symbols. 
      /* Single tuple */
      additive-symbols: 3 "*";

      additive-symbols:
        3 "0",
        2 "\2E\20",
        1 url(symbol.png);
    • negative - указывает символы, которые следует добавить в конец или начало представления счетчика, если значение отрицательное. 
      /* One value */
      negative: "--"; /* Adds '--' before if counter value is negative */

      /* Two values */
      negative: "(" ")"; /* Adds '(- before and ')' after if value is negative */
    • prefix - указывает символ, который должен быть добавлен к представлению маркера. Префиксы добавляются к представлению на последнем этапе, перед любыми символами, добавляемыми к отрицательным значениям свойства negative. 
      /* value: string, image, or identifier */
      prefix: "§";
      prefix: "Page ";
      prefix: url(bullet.png);
    • suffix - указывает символ, который добавляется после представления маркера, включая любые символы, добавленные к отрицательным значениям свойства negative. 
      /* value: string, image, or identifier */
      suffix: "!";
      suffix: " page.";
      suffix: url(bullet.png);
    • range - определяет диапазон значений, к которым применим стиль счетчика. Если стиль счетчика используется для представления значения счетчика за пределами диапазонов, определенных этим дескриптором, стиль счетчика вернется к своему резервному стилю, описанному в свойстве fallback. 
      range: 2 5;
      range: infinite 10;
      range: 6 infinite;
      range: infinite infinite;
    • pad - определяет минимальную длину счетчика. Если представление маркера короче указанной длины, то представление маркера будет дополнено указанным символом заполнения; остальные представления отображаются без дополнительного заполнения. 
      pad: 3 "0";
      pad: "+" 5;
    • speak-as - описывает, как экранные дикторы, должны объявлять стиль счетчика. Например, значение маркера элемента списка может быть прочитано как числа или буквы алфавита для упорядоченных списков или как звуковые подсказки для неупорядоченных списков.
      • auto - значение по умолчанию (настройки браузера)
      • bullets - будет зачитана фраза или звуковая подсказка, определенная для неупорядоченного элемента списка.
      • numbers - числовое значение счетчика будет озвучено на языке документа.
      • words - будет озвучено строковое значение счетчика на языке документа.
      • spell-out - будет читать значение счетчика по буквам. Если пользовательский агент не знает, как читать определенный символ счетчика - озвучит как будто значение было числом.
    • fallback - указывает имя счетчика системы, к которому следует вернуться, если указанная система не может построить представление значения счетчика или если значение счетчика выходит за пределы указанного диапазона. Если резервный счетчик также не может представить значение, то используется резерв этого счетчика, если он указан. Если нет описанных резервных счетчиков или если цепочка резервных систем не может представить значение счетчика, то в конечном итоге будет выполнен возврат к десятичному стилю. 
      fallback: lower-alpha;
