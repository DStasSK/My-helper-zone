Формат Web Video Text Tracks (WebVTT)это формат для отображения синхронизированных текстовых треков (такие как субтитры или подписи) с помощью элементов track. Основная цель файлов WebVTT — добавить текстовые наложения к элементам video. WebVTT является текстовым форматом, который должен быть закодирован с использованием UTF-8 (en-US). В этих файлах вы можете использовать пробелы и табы для отступов.

MIME тип файлов WebVTT — text/vtt.

Структура файла с титрами (.vtt).

Структура WebVTT состоит из следующих компонентов, некоторые из которых являются необязательными, в следующем порядке:

  • Необязательный знак порядка байтов (BOM).
  • Строка "WEBVTT".
  • Дополнительный текстовый заголовок справа от "WEBVTT" (должен быть хотя бы один пробел, кроме перевода строки или "-->".).
  • Пустая строка, которая эквивалентна двум последовательным переводам строки.
  • Ноль или более реплик или комментариев.
  • Ноль или более пустых строк.
WEBVTT - Этот файл содержит реплики.

00:01:14.815 --> 00:01:18.114
- Что?
- Где мы сейчас?

00:01:18.171 --> 00:01:20.991
- Это большая страна летучих мышей.

Каждая реплика начинается с метки времени, которое определяет начало отображения нижележащего текста, далее идут символы --> и вторая метка времени, которое определяет прекращения отображения связанного текста. После метки времени идут одна или несколько строк, начинающихся с дефиса (-), каждая из которых содержит часть текстовой дорожки для отображения.

Мы также можем разместить комментарии в нашем файле .vtt, чтобы помочь нам запомнить важную информацию о частях нашего файла. Они должны быть в отдельных строках, начинающихся со слова NOTE.

Важно не использовать дополнительные пустые строки в реплике, например, между строкой синхронизации и текстом реплики. WebVTT использует строчные разделители, поэтому пустая строка обозначит окончание реплики.

WEBVTT

NOTE комментарий №1.

NOTE комментарий
№2.

NOTE
комментарий
№3.

Стилизация титров.

Текст, идущий в титрах к видео, можно стилизовать с теми или иными целями. В основном это делаестся для более удобного прочтения, например могут задавать цвет фона, что позволит видеть текст в независимости от цвета картинки видео (например, если цвет кадра видео совпадет с цветом текста, то прочитать текст будет затруднительно).

Для стилизации текста титров доступны следующие CSS-свойства:
  • background и его свойства
  • color
  • font и его свойства
  • line-height
  • opacity
  • outline
  • ruby-position
  • text-combine-upright
  • text-decoration
  • text-shadow
  • visibility
  • white-space

Параметры со стилями текста могут нахожиться как в нутри файла с титрами, так и в отдельном CSS файле.

Стилизация титров внутри файла с титрами.

При стилизации титров внитри файла сс титрами есть ряд ограничений:
  • блок с параметрами начинается со слов STYLE, сами параметры идут со следующей строки.
  • параметры стилей, находящиеся в блоке, не должны разделять пустой строкой.
  • блок со стилями должен идти до текста титров, не после.
  • блоков со стилями может быть несколько, каждый должен начинаться со слов STYLE.
  • блоки со стилями могут быть разделены комментариями (блок NOTE).
  • если нужно стилизовать отдельную текстовую дорожку, то стили можно располагать после временных интервалов (параметры).
  • ...
WEBVTT

STYLE
::cue {
  background-image: linear-gradient(to bottom, dimgray, lightgray);
  color: papayawhip;
}

NOTE комментарий.

STYLE
::cue(b) {
  color: peachpuff;
}

00:00:00.000 --> 00:00:10.000
- Hello <b>world</b>.

00:00:11.000 --> 00:00:14.000 position:10%,line-left align:left size:35%
Where did he go?

00:00:16.000 --> 00:00:18.500 position:90% align:right size:35%
I think he went down this lane.

00:00:05.000 --> 00:00:10.000 line:0 position:20% size:60% align:start
Some time ago in a place rather distant....

Стилизация титров во внешнем CSS файле.

Для стилизации текста титров во внешнем CSS файле используют Псевдоэлемент ::cue, который соответствует репликам WebVTT в выбранном элементе. Он может быть использован для задания стилей титров и других реплик в медиа с VTT треками.

Заданные в CSS файле свойства применяются ко всему набору реплик, как если бы они были единым целым. Единственным исключением является background и его сокращённые свойства, которые применяются к каждой реплике отдельно, чтобы не создавать блоки, которые занимают неожиданно большие области медиа.

::cue {
  color: #fff;
  font-weight: bold;
  background-color: rgba(0, 0, 0, 0.6);
}

video::cue(b) {
  background-image: linear-gradient(to bottom, dimgray, lightgray);
  color: peachpuff;
}
текст, выделенный жирным шрифтом с помощью элемента b, имеет цвет peachpuff и заданный цвет фона в виде градиента.
