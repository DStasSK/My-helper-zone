Атрибут http-equiv элемента meta

Атрибут http-equiv предназначен для конвертирования метатега в заголовок HTTP. Этот атрибут определяет прагму, которая может изменять поведение серверов и пользователей. Значение прагмы определяется с помощью атрибута content.

Браузеры преобразовывают значение атрибута http-equiv, заданное с помощью content, в формат заголовка ответа HTTP и обрабатывают их, как будто они прибыли непосредственно от сервера.

Когда атрибут http-equiv указан в элементе meta, элемент meta является прагма директивой, которая предоставляет дополнительную информацию о документе.

<meta http-equiv="content-security-policy | content-type |
                  default-style | x-ua-compatible | refresh"
      content="values">
  • content-security-policy - позволяет авторам страниц определять политику контента для текущей страницы. Политики контента в основном определяют разрешенные серверные источники и конечные точки скриптов, которые помогают защититься от атак с использованием межсайтового скриптинга.
    См. также Content-Security-Policy.
  • content-type - (устаревшее) oбъявляет тип MIME и кодировку символов документа. Атрибут content должен иметь значение "text/html; charset=utf-8", если указано. Это эквивалентно элементу meta с указанным атрибутом charset и имеет то же ограничение на размещение в документе.
    Mожет использоваться только в документах, обслуживаемых с text/html — не в документах, обслуживаемых с типом XML MIME.
    См. также content-type.
  • default-style - задает имя (через атрибут name) набора таблиц стилей CSS по умолчанию. Специализация этой прагмы - предпочтительный стиль таблиц, используемый на странице. Aтрибут content должен содержать заголовок элемента link или style, который содержит CSS таблицу стилей.
  • x-ua-compatible - Если указано, атрибут content должен иметь значение "IE=edge", то агенты пользователей должны игнорировать эту прагму.
  • refresh - эта инструкция определяет количество секунд до перезагрузки страницы или перехода на другую страницу. В качестве значения принимает два параметра из атрибута content: неотрицательное целое число, определяющее таймер перехода и допустимый URL для перехода на другую страницу. Таймер запускается, когда страница полностью загружена, то есть после срабатывания событий load и pageshow. 
    <meta http-equiv="refresh" content="5; url=https://example.com/">
Устаревшие значения прагм:
content-language Позволяет указать основной язык для всего документа. 
<!-- устаревшая форма определения - не использовать -->
<meta http-equiv="content-language" content="ru" />

<!-- В HTML5 указание языка упрощено: -->
<html lang="ru">
expires Устанавливает дату и время, после которой информация в документе будет считаться устаревшей.
pragma Способ кэширования документа.
Cache-Control

Управление кэшированием. Должно давать четкие указанию браузеру и промежуточным серверам как вести кэширование этой страницы. Инструкции не чувствительны к регистру и имеют необязательный аргумент, который может быть указан как в кавычках, так и без них. Несколько инструкций разделяются запятыми.

  • public - документ кэшируется во всех доступных кэшах
  • no-store - может быть кэширован, но не сохраняется в архиве
  • max-age=time, must-revalidate - указывает браузеру сколько секунд хранить документ в кэше
  • private - кэшируется браузером, не кэшируется proxy-сервером
  • no-cache - запрет на кэширование (браузер и proxy-сервер документ не кэшируют)
  • max-age=time, proxy-revalidate - указывает сколько секунд хранить документ в кэше proxy-серверу
<!-- для страниц типа результатов поиска -->
<meta http-equiv="Cache-Control"
         content="no-cache, no-store, must-revalidate">
<!-- для публичных страниц типа статей блога -->
<meta http-equiv="Cache-Control"
         content="public, max-age=31536000">
Полный список возможных значений. 
<meta http-equiv="Content-Type" content="text/html; charset=utf-8">
<meta http-equiv="expires" content="Sun, 01 Jan 2013 07:01:00 GMT">
<meta http-equiv="refresh" content="300"> <!-- обновление каждые 5 минут -->
<meta http-equiv="refresh" content="20; url=page4.html"> <!-- переход по url -->
<meta http-equiv="default-style" content="default">
