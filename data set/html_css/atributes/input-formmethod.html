Атрибут formmethod элемента input.

Атрибут formmethod сообщает браузеру, каким методом следует передавать данные формы на сервер. Данный атрибут можно применять к элементу input с атрибутом type имеющим значение submit или image а также к элементу button. Значением атрибута является строка, указывающая метод HTTP, который следует использовать при отправке данных формы; это значение переопределяет любой атрибут метода, заданный в форме-владельце. Значение по умолчанию - get.

<input type="submit" formmethod="get | post">
<input type="image" formmethod="get | post">
get

URL-адрес создается путем начала URL-адреса, заданного атрибутом formaction или action, добавления символа вопросительного знака ("?"), а затем добавления данных формы, закодированных, как описано в атрибуте formenctype или enctype формы. Затем этот URL-адрес отправляется на сервер с помощью HTTP-запроса get. Этот метод хорошо работает для форм, которые содержат только символы ASCII и не имеют побочных эффектов. Это значение по умолчанию.

Этот метод передачи данных является одним из самых распространённых и предназначен для получения требуемой информации и передачи данных в адресной строке. Пары «имя=значение» присоединяются в этом случае к адресу после вопросительного знака и разделяются между собой амперсандом (символ &). Полный адрес к примеру будет:

http://site.ru/doc/?name=Vasya&password=pup
Объём данных в методе ограничен 4 Кб.
post Метод передачи данных post посылает на сервер данные в запросе браузера, объём пересылаемых данных ограничен лишь настройками сервера. Данные формы включаются в тело запроса, который отправляется на URL, указанный атрибутом formaction или action, с использованием метода HTTP post. Этот метод поддерживает сложные данные и вложения файлов.
dialog Предназначен для закрытия диалогового окна, когда форма располагается внутри элемента dialog. Этот метод используется для указания того, что кнопка закрывает диалоговое окно, с которым связан ввод, и вообще не передает данные формы.
