Атрибут enterkeyhint

Атрибут enterkeyhint — это перечисляемый атрибут, определяющий, какую метку действия (или значок) следует отображать для клавиши Enter на виртуальных клавиатурах. Это позволяет изменить поведение полей формы и дать пользователю подсказку, что ему ожидать при работе с клавиатурой. Если атрибут enterkeyhint не указан, браузер самостоятельно подбирает наиболее подходящий вид кнопки (например - из атрибутов inputmode, type или pattern).

<input enterkeyhint="enter | done | go | next | previous | search | send">
  • enter - обычно вставляется новая строка. return, ↵
  • done - нажатие Enter не будет отправлять форму, а только завершит ввод в текущем поле и свернёт клавиатуру. done, ✅
  • go - нажатие Enter отправит форму, даже если поле ввода многострочное. go, 🡢, →
  • next - перенаправляет к следующему полю ввода текста. next, ⇥
  • previous - переход к предыдущему полю ввода. return, ⇤
  • search - обычно перенаправляет пользователя к результатам поиска по введенному им тексту. Использует Enter для поиска или отправки формы. search, 🔍
  • send - нажатие Enter выполнит отправку формы. send, ➣

Элементы управления формами (например, элементы textarea или input) или элементы, использующие contenteditable, могут указывать атрибут inputmode для управления тем, какой тип виртуальной клавиатуры будет использоваться. Для дальнейшего улучшения пользовательского опыта можно настроить клавишу Enter, указав атрибут enterkeyhint, указывающий, как должна быть обозначена клавиша Enter (или какой значок должен быть показан). Клавиша Enter обычно представляет, что пользователь должен сделать дальше; типичные действия: отправка текста, вставка новой строки или поиск.
