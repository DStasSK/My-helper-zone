Атрибут aria-label

Атрибут aria-label обеспечивает альтернативное текстовое описание для элементов интерфейса, которые визуально понятны, но могут быть проблематичны для восприятия пользователями скринридеров. Цель атрибута - добавить текстовую подсказку к элементу, с помощью которой пользователи скринридера поймут его назначение. Этот текст не отображается визуально, но читается вспомогательными технологиями. Атрибут aria-label необходим для того, чтобы пользователи программ чтения с экрана понимали его функцию

Примечание: атрибут aria-label предназначен для описания элементов, где неявная или явная роль не запрещает именование. Настоятельно рекомендуется отдавать приоритет использованию aria-labelledby перед aria-label, если существует видимая метка, на которую элемент может ссылаться (через id) и от которой может получать свое описание.

По умолчанию доступное имя кнопки - это содержимое между открывающим и закрывающим тегами button, доступное имя изображения — это содержимое его атрибута alt, а доступное имя поля ввода формы — это содержимое связанного с ним элемента label. Если ни один из этих вариантов недоступен или если доступное имя по умолчанию не подходит, используйте атрибут aria-label для определения доступного описания элемента.

Хотя атрибут aria-label можно использовать для любого элемента, имеющего доступное имя, на практике он поддерживается только для интерактивных элементов, виджетов, ориентиров, изображений и iframe.

Оптимальная метка aria-label должна быть:
  • Содержательной — объясняющей функцию элемента
  • Краткой — идеально от 1 до 5 слов
  • Конкретной — избегайте расплывчатых описаний
Атрибут aria-label не поддерживается в тегах: caption, code, definition, deletion, emphasis, generic, insertion, mark, paragraph, presentation / none, strong, subscript, suggestion, superscript, term, time.
