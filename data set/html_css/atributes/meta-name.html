Атрибут name элемента meta

Атрибут name Устанавливает название/имя метаданных документа. Атрибуты name и content могут использоваться вместе для предоставления метаданных документа в виде пар «имя-значение», при этом атрибут name задает имя метаданных, а атрибут content задает значение.

Одновременно использовать атрибуты name и http-equiv в одном тэге meta не допускается.
<meta name="property" content="myValues">
Стандартные имена метаданных (часть):
application-name имя приложения, запущенного на веб-странице. Браузеры могут использовать это для идентификации приложения. Он отличается от элемента title, который обычно содержит имя приложения, но может также содержать информацию, например имя документа или статус.
Отдельные веб-страницы не должны определять этот атрибут.
author Имя автора документа. Значение должно быть строкой произвольной формы с указанием имени одного из авторов страницы.
description Описание текущего документа. Значение должно быть строкой произвольной формы, описывающей страницу. Значение должно быть подходящим для использования в каталоге страниц, например, в поисковой системе.
generator

Значение должно быть строкой произвольной формы, которая идентифицирует один из пакетов программного обеспечения, использованных для создания документа. Это значение не должно использоваться на страницах, разметка которых не создается программным обеспечением, например на страницах, разметка которых была написана пользователем в текстовом редакторе.

<meta name="generator" content="WordPress 5.2" />
keywords

Список ключевых слов, встречающихся на странице. Значение должно быть набором разделенных запятыми ключевых слов, относящихся к странице. Многие поисковые системы не рассматривают такие ключевые слова, потому что эта функция исторически использовалась ненадежно и спамила результаты поиска.

<meta name="keywords" content="слова,для,поисковых,систем" />
viewport Дает подсказки о размере начального размера области просмотра. Управляет просмотром сайта на мобильных устройствах; пример. 
<meta name="viewport" content="width=device-width">
значения атрибута content для meta name="viewport"
referrer Kонтролирует содержимое HTTP. Referer HTTP - заголовок, прикреплённый к любому запросу, отправленному из этого документа. Позволяет отслеживать перемещения пользователей между страницами; пример.
theme-color Указывает предлагаемый цвет, который пользовательские агенты должны использовать для настройки отображения страницы или окружающего пользовательского интерфейса. Атрибут content содержит допустимый цвет CSS. Атрибут media с допустимым списком запросов media может быть включен для установки носителя, к которому применяются метаданные цвета темы. 
<meta name="theme-color" content="#50a2de"
      media="(prefers-color-scheme: light)">
<meta name="theme-color" content="#1f2d3b"
      media="(prefers-color-scheme: dark)">

color-scheme

Указывает одну или несколько цветовых схем, с которыми совместим документ. Браузер будет использовать эту информацию совместно с настройками браузера или устройства пользователя, чтобы определить, какие цвета использовать для всего: от фона и переднего плана до элементов управления формами и полос прокрутки. Основное применение — указать совместимость со светлыми и темными цветовыми режимами и порядок их предпочтения.

Значение свойства content для color-scheme может быть одним из следующих:
  • normal - Документ не поддерживает цветовые схемы и должен отображаться с использованием цветовой палитры по умолчанию.
  • light, dark, light dark, dark light - Одна или несколько цветовых схем, поддерживаемых документом. Указание одной и той же цветовой схемы более одного раза имеет тот же эффект, что и указание ее только один раз. Указание нескольких цветовых схем означает, что первая схема предпочтительна для документа, но что вторая указанная схема приемлема, если пользователь предпочитает ее.
  • only light - Указывает, что документ поддерживает только светлый режим со светлым фоном и темными цветами переднего плана. Согласно спецификации, только темный режим недопустим, поскольку принудительное отображение документа в темном режиме, когда он на самом деле с ним несовместим, может привести к нечитаемому контенту; все основные браузеры по умолчанию используют светлый режим, если не настроено иное.
<meta name="color-scheme" content="dark light" />
url

Прекращает индексацию страницы поисковой системой, и перенаправляет робота поисковой машины по указанной ссылке. Применяется для отмены индексации «зеркала» и генерируемых страниц.

<meta name="url" content="http://ru.wikipedia.org/" />

Нестандартные значения атрибута

application-title Используется для настройки заголовка приложения для веб-приложений, установленных как отдельные приложения на поддерживающих настольных операционных системах. Хотя текстовое содержимое элемента title обычно отображается во вкладках браузера, когда приложение запущено в браузере, имя метаданных application-title может использоваться для установки другого заголовка для приложения, когда оно запущено как отдельное установленное приложение.
Слабая поддержка даже современными браузерами.
creator определят в свободном формате имя создателя документа. Это также может быть имя института. Если же имён больше чем одно, то должны быть использованы несколько элементов meta.
googlebot синоним robots, но только следует за Googlebot, сканирует индексы для Google.
publisher определяет в свободном формате имя того, кто опубликовал документ. Это также может быть имя института.
robots

Значения content для meta name="viewport"

width device-width или целое положительное число

Устанавливает ширину области просмотра в пикселях.

device-width соответствует 100vw 
<meta name="viewport" 
      content="width=device-width">
height device-height или целое положительное число

Устанавливает высоту области просмотра в пикселях.

device-height соответствует 100vh
initial-scale Число от 0.0 до 10.0 Устанавливает соотношение между шириной устройства (device-width в портретном режиме или device-height в ландшафтном режиме) и размером области просмотра. 
<meta name="viewport" 
      content="initial-scale=1">
maximum-scale Число от 0.0 до 10.0 Задаёт максимальное значение масштаба. Должно быть больше или равно minimum-scale, в противном случае игнорируется.
minimum-scale Число от 0.0 до 10.0 Задаёт минимальное значение масштаба. Должно быть меньше или равно maximum-scale, в противном случае игнорируется.
user-scalable yes или no Если указано no, то пользователь не сможет масштабировать веб-страницу.
По умолчанию используется yes.
viewport-fit auto, contain или cover
  • auto не влияет на начальный макет области просмотра, и вся веб-страница видна.
  • contain означает, что область просмотра масштабируется для заполнения самого большого прямоугольника, вписанного в дисплей.
  • cover означает, что область просмотра масштабируется для заполнения дисплея устройства. Настоятельно рекомендуется использовать переменные вставки безопасной области, чтобы гарантировать, что важный контент не окажется за пределами дисплея.

Значения content для meta name="robots"

index Разрешает роботу индексировать данную страницу. 
<meta name="robots" content="index,follow" />
<meta name="robots" content="all" />
noindex Запрещает роботу индексировать текущую страницу. Она не попадает в базу поисковика и её невозможно будет найти через поисковую систему. 
<meta name="robots" content="noindex" />
follow Разрешает роботу переходить по ссылкам на данной странице. 
<meta name="robots" content="follow" />
nofollow Запрещает роботу переходить по ссылкам на данной странице. При этом всем ссылкам не передаётся ТИЦ (тематический индекс цитирования) и PageRank. 
<meta name="robots" content="nofollow" />
all эквивалентно значению index, follow, используется только в Google.
none эквивалентно значению noindex, nofollow, используется только в Google.
noarchive Запрещает роботу кэшировать данную страницу. 
<meta name="robots" content="noarchive" />
используется в Google, Yahoo, Bing.
nocache Синоним noarchive, используется Bing.
nocache Запрещает отображение любого описания страницы на странице результатов поиска; используется Google, Bing.
noimageindex Запрещает отображение этой страницы в качестве ссылающейся страницы индексированного изображения; используется Google.
  • Эти правила соблюдают только кооперативные роботы. Не ожидайте, что они предотвратят сборщиков электронной почты.
  • Роботу все равно нужно получить доступ к странице, чтобы прочитать эти правила. Чтобы предотвратить потребление полосы пропускания, подумайте, не лучше ли использовать файл robots.txt.
  • Элемент meta name="robots" и файл robots.txt служат разным целям: robots.txt управляет сканированием страниц и не влияет на индексацию или другое поведение, контролируемое метаданными robots. Страница, которую невозможно просканировать, все равно может быть проиндексирована, если на нее ссылается другой документ.
  • Если вы хотите удалить страницу, noindex сработает, но только после того, как робот снова посетит страницу. Убедитесь, что файл robots.txt не предотвращает повторные посещения.
  • Некоторые значения являются взаимоисключающими, например index и noindex или follow и nofollow. В этих случаях поведение робота не определено и может различаться между ними. Некоторые поисковые роботы, такие как Google, Yahoo и Bing, поддерживают те же значения для HTTP-заголовка X-Robots-Tag; это позволяет использовать эти правила в документах, отличных от HTML, например, в изображениях.
