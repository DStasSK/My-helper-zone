Атрибут charset элемента meta

Этот атрибут объявляет кодировку символов документа. Если атрибут присутствует, его значение должно быть нечувствительным к регистру ASCII-соответствием для строки "utf-8", поскольку UTF-8 является единственной допустимой кодировкой для документов HTML5. Элементы meta, объявляющие кодировку символов, должны быть расположены полностью в пределах первых 1024 байт документа.

Помимо UTF-8 атрибут может принимать другое стандартное имя IANA MIME для кодировки символов документа.
<meta charset="utf-8">

<!-- аналогично устаревшему -->
<meta http-equiv="Content-Type" content="text/html; charset=IANAcharset">
  • Не следует использовать ASCII-несовместимые кодировки, чтобы избежать угроз безопасности: браузеры, не поддерживающие их, могут интерпретировать вредоносный контент как HTML. Это относится к семейству кодировок JIS_C6226-1983, JIS_X0212-1990, HZ-GB-2312, JOHAB и EBCDIC. ASCII-несовместимые кодировки - это те, которые не преобразуют 8-битные коды точек 0x20, 0x7E, 0x0020, 0x007E в коды Unicode точек.
  • Авторы не должны использовать кодировки CESU-8, UTF-7, BOCU-1 и/или SCSU, так как есть примеры атак межсайтового скриптинга использующих данные кодировки.
  • Авторам не следует использовать кодировку UTF-32, потому что не все алгоритмы кодирования HTML5 могут отличить её от UTF-16.
