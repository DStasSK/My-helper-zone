Элемент <video> Добавляет, воспроизводит и управляет настройками видеоролика на веб-странице. Путь к файлу задаётся через атрибут src или вложенный элемент <source>. Список поддерживаемых браузерами аудио и видеокодеков ограничен и приведён в таблице. Для универсального воспроизведения в указанных браузерах видео кодируют с помощью разных кодеков и добавляют файлы одновременно.

<video>
  <source src="адрес">
</video>
Также видео можно отображать на странице при помощи тэга <embed>.

Кодеки и браузеры

Если MP4 содержит видео H.264 (базовый или основной уровень сложности) и аудио AAC, Chrome сможет его воспроизвести. Если MP4 содержит что-то более сложное (например, видео H.265/HEVC и более сложный аудиоформат), у Chrome нет ни единого шанса его воспроизвести.

Chrome также будет воспроизводить видеофайлы WebM, если они содержат видео VP9 (стандарт Youtube) и аудио Opus.

Любая другая комбинация видео+аудио+контейнер не сработает, если вы не установите какие-нибудь плагины/дополнения, которые добавят дополнительные функции в ваш браузер.

Атрибуты

autoplay Видео начинает воспроизводиться автоматически после загрузки страницы. 
<video autoplay="autoplay"</video>
атрибут autoplay не работает без muted в хроме.
controls Указывает браузеру, что нужно отобразить базовые элементы управления воспроизведением (воспроизведение, пауза, громкость). 
<video controls></video>

Для отключения стандартных элементов управления нужно удалить атрибут и задать стили в CSS.

<video controls="false"></video>

/* css */
video::-webkit-media-controls {
  display: none! important;
}

/* указываем для отдельных префиксов */
video::-moz-media-controls {}
video::-o-media-controls {}
video::-ms-media-controls {}
video::-webkit-media-controls-fullscreen-button {}
video::-moz-media-controls-fullscreen-button {}

/* отключает отдельные кнопки */
video::-webkit-media-controls-play-button {}
video::-webkit-media-controls-volume-slider {}
video::-webkit-media-controls-mute-button {}
video::-webkit-media-controls-timeline {}
video::-webkit-media-controls-current-time-display {}

Для отключения возможности включать встроенное управление можно задать css свойство pointer-events: none, но при этом клики кнопки мыши не будет работать - видео будет не управляемым и может нарушиться логика JS, если она есть.

Для предотвращения появления встроенных кнопок управления в полном экране: можно не разварачивать видео на полный экран, а разворачивать контейнер с видео.
Удаление controls в разметке HTML5 убивает видео на iPad - можно использовать JS? 
window.onload = ()=> { //после загрузки
  video = document.querySelector('my_selector');
  video.removeAttribute('controls');
}
controlslist Указывает, какие элементы управления отображать для видеоэлемента, когда указан атрибут controls. 
<video controls controlslist="
  nodownload | nofullscreen | noremoteplayback">
</video>
  • nodownload - отключить кнопку загрузки
  • nofullscreen - откл. кнопку перехода в полный экран
  • noremoteplayback - нет удаленного воспроизведения
данный атрибут не поддерживается в:
FF, Safari, WebView on iOS.
disablePictureInPicture Свойство указывает, отключена ли функция «картинка в картинке» для текущего элемента. 
<video disablePictureInPicture="true"></video>
<video disablePictureInPicture></video>
<video disablePictureInPicture="false"></video>
height Задаёт высоту области для воспроизведения видеоролика, возможные значения: px или %. 
<video height="значение"></video>
loop Повторяет воспроизведение видео с начала после его завершения. 
<video loop="loop"></video>
muted Отключает звук при воспроизведении видеоролика. 
<video muted="muted"></video>
Некоторые браузеры (Например Chrome 70.0) не могут автоматически проигрываться если <video> имеет атрибут muted.
playsinline Экраны устройств использующих Android или iOS довольно маленькие, поэтому при просмотре video на браузере данных устройств, оно автоматически воспроизводит видео в полноэкранном режиме (Fullscreen). Атрибут playsinline не даёт браузеру воспроизводить видео в полноэкранном режиме. 
<video playsinline></video>
poster Указывает адрес картинки, которая будет отображаться, пока видео не доступно или не воспроизводится. Если атрибут не задан, то будет отображаться первый кадр видеофайла. 
<video poster="адрес"></video>
preload Атрибут, отвечающий за предварительную загрузку видеоконтента. Не является обязательным, некоторые браузеры игнорируют его. Этот атрибут игнорируется, если установлен атрибут autoplay. Пустое значение атрибута воспринимается как auto. Значение по умолчанию - none. 
<video preload="none | metadata | auto"></video>
  • none - не загружать предварительную информацию об файле
  • metadata - загрузить только служебную информацию (продолжительность и др.)
  • auto - загрузить файл целиком при загрузке страницы
src Указывает путь к воспроизводимому файлу. 
<video src="адрес"></video>
width Задаёт ширину области для воспроизведения видеоролика, возможные значения: px или %. 
<video width="значение"></video>

Пример оформления.

<video width="400" height="300" controls="controls" poster="video/jane.jpg">
  <source src="video/jane.ogv" type='video/ogg; codecs="theora, vorbis"'>
  <source src="video/jane.mp4" type='video/mp4; codecs="avc1.42E01E, mp4a.40.2"'>
  Элемент video не поддерживается вашим браузером.
  <a href="video/jane.mp4">Скачайте видео</a>
</video>

Установка начала воспроизведения с нужного времени.

Подскажите, как задать воспроизведение видео с определенной секунды.
Установить значению свойства элемента видео - currentTime необходимое значение в секундах.
<video src="путь"
   id="player" controls width="400" height="300"
   crossorigin="anonymous">
</video>

<script>
  let timeToPlay = 2;
  let video = document.querySelector('#player');
  video.onplay = (el) => {
    el.srcElement.currentTime = timeToPlay; // текушее время видео
    el.srcElement.play();
    timeToPlay = 0;
  }
</script>
