Псевдоэлементы

::after & ::before ::backdrop ::first-letter ::first-line ::marker ::selection ::grammar-error ::spelling-error

::before и ::after

Данные псевдоэлементы позволяют добавлять контент до или после блоков, которым задан данный элемент. Кроме того, элементы, такие как <input>, <textarea> или <img> не содержат в себе другие элементы, таким образом псевдо-элементы к ним не применяются.

Псевдоэлементы ::before и ::after работает совместно со свойством content.
По умолчанию ::before и ::after создают строчный элемент.
перенос строки через псевдоэлемент ::after 
span::after {
  content: '\A';
  white-space: pre;
}

::backdrop

Псевдоэлемент отображается ниже самого верхнего элемента в стеке по оси Z, но выше всех остальных элементов на странице, если они имеются. Обычно ::backdrop применяется для затемнения страницы, чтобы акцентировать внимание на фотографии или диалоговом окне, которые выводятся поверх такого затемнения.

В настоящее время работает только с модальными окнами, созданными с помощью элемента dialog.
.elem::backdrop { ... }
Применяется со скриптами. Например: 

Например: <button id="openDialog">Вход на сайт</button>

<dialog id="exampleDialog">
  <form id="authentication" action="handler.php">
    <p><input name="login" required autofocus placeholder="Логин"></p>
    <p><input type="password" name="pass" required placeholder="Пароль"></p>
    <p>
      <button type="submit">Войти</button>
      <button id="closeDialog">Закрыть окно</button>
    </p>
  </form>
</dialog>

<script>
  let dialog = document.querySelector('#exampleDialog');
  document.querySelector('#openDialog').onclick = function() {dialog.showModal();}
  document.querySelector('#closeDialog').onclick = function() {dialog.close();}
</script>
Браузер Internet Explorer, а также Edge с версии 12 до 18 включительно поддерживают псевдоэлемент ::-ms-backdrop.
Браузер Chrome до версии 37 и браузер Opera до версии 24 поддерживают псевдоэлемент ::-webkit-backdrop.

::first-letter

Определяет стиль первого символа в тексте элемента, к которому добавляется. К этому псевдоэлементу могут применяться только стилевые свойства, связанные со свойствами шрифта, полями, отступами, границами, цветом и фоном.

p::first-letter { ... }
Hе работает на элементах со значением display:inline;.
Ставим display:block; или display:inline-block;

::first-line

Задаёт стиль первой строки форматированного текста. Длина этой строки зависит от многих факторов, таких как используемый шрифт, размер окна браузера, ширина блока, языка и т. д. В правилах стиля допустимо использовать только свойства, относящиеся к шрифту, изменению цвета текста и фона.

p::first-line { color: red }

Lorem ipsum dolor sit amet consectetur adipisicing elit. Qui laboriosam delectus aperiam rem, hic repellat natus dignissimos magni vero eveniet deserunt voluptatibus soluta illum. Esse mollitia animi, iste obcaecati doloremque laboriosam praesentium eaque fuga labore, fugiat ipsa amet ad neque.

::marker

Данный псевдоэлемент позволяет устанавливить вид маркера через свойство content, а также управлять стилем маркера (менять его цвет или параметры шрифта). ::marker работает для элементов li, summary, а также для элементов, у которых свойство display:list-item.

С помощью псевдоэлемента ::marker допустимо устанавливать любые свойства шрифта, а так же свойства white-space, color, content, анимацию и переходы.

  • 1
  • 2
  • 3
<ul class="list">
   <li>1</li>
   <li>2</li>
   <li>3</li>
</ul>

<style>
   .list ::marker {color: red}
   .list li:nth-child(2)::marker {content:'+'}
   .list li:nth-child(3)::marker {content:'+  '}
</style>
Браузер Safari, а также мобильный Firefox, не поддерживают свойства анимации и переходов.

::selectionПозваляет задать цвет выделения текста.

Выделение обычно применяется для копирования в буфер фрагмента текста, чтобы после этого вставить выделенный фрагмент в другое место. Например, в текстовый редактор для дальнейшей правки текста. Сам цвет и фон выделенного фрагмента определяется настройками операционной системы, но с помощью CSS эти параметры можно задать самостоятельно.

В правилах стилей допускается использовать следующие свойства: color, background, background-color, cursor, outline и text-shadow.

Firefox до версии 62 использует нестандартный псевдоэлемент ::-moz-selection.
.elem ::selection {
   color:#007;
   background-color:#eef;
}

::grammar-error

Псевдоэлемент ::grammar-error применяет стиль к фрагменту текста, который браузер помечает как написанный с грамматической ошибкой. Сама проверка грамматики выполняется браузером на основе указанного языка и встроенного словаря.

В правилах стилей допускается использовать следующие свойства: color, background-color, cursor, caret-color, outline, text-decoration, text-emphasis-color и text-shadow.

::spelling-error

Псевдоэлемент ::spelling-error применяет стиль к фрагменту текста, который браузер помечает как написанный с орфографической ошибкой. Сама проверка правописания выполняется браузером на основе указанного языка и встроенного словаря.

В правилах стилей допускается использовать следующие свойства: color, background-color, cursor, caret-color, outline, text-decoration, text-emphasis-color и text-shadow.

Псевдоклассы для оформления ссылок.

:active :link :visited :any-link :target
:active

Псевдокласс :active устанавливает стиль оформления ссылки в момент нажатия на неё. Чтобы пользователь понимал, что его щелчок по элементу сработал, в момент щелчка меняют вид элемента, для этого используется псевдокласс :active, который добавляется к селектору. В основном применяется к ссылкам и кнопкам, но ими не ограничен.

a {color: #009}
a:active {color: #f30}
:link

Этот параметр определяет необходимое стилевое оформление ссылкам, которые ещё не посещались. Благодаря псевдоклассу можно изменить стиль ссылки и сайта в целом установив для неё необходимый формат.

<a href="#a">которые ещё не посещались</a>

#link a{color: #009}
#link a:link{color: #963}
#link a:active{color: #f00}

#link a:link:after{content:' (перехода не было)'}
#link a:active:after{content:' (ссылка нажата)'}
:visited

Устанавливает стиль ссылки по которой был осуществлён переход. При использовании данного параметра, происходит замена стилей в соответствии с выбранными параметрами.

Псевдоэлемент :visited признан небезопасным псевдоэлементом и потому может быть кастомизирован очень ограниченным количеством свойств: color, background-color, border-*-color, outline-color и параметр цвета атрибутов fill и stroke у SVG изображений. Кроме того, для перечисленного списка свойств нельзя установить цвета через rgba() или hsla(); или задать им значение transparent.
<a href="#a">по которой был осуществлён переход</a>

#link a{color: #009}
#link a:link{color: #963}
#link a:visited{color: #a0d}
#link a:active{color: #f00}

#link a:link:after{content:' (перехода не было)'}
#link a:visited:after{content:' (переход был)'}
#link a:active:after{content:' (ссылка нажата)'}
Из примера видно, что после перехода по ссылке параметр
a:link:after {content:' (перехода не было)'}
не сбросился. Это происходит по причинам безопасности.

Представленные выше три псевдокласса для оформления стилей могут переопределять друг друга, чтобы избежать этого необходимо их указывать в определенном порядке, а именно:

a:link, a:visited, a:hover, a:active, a:focus {}
:any-link

Псевдокласс :any-link используется для выбора любых элементов, которые являются ссылками. При этом ссылка может быть как посещённой, так и не посещённой. Обычно :any-link применяется для тех ссылок, которые не должны менять свой цвет после посещения, например, ссылки меню.

Псевдокласс :any-link эквивалентен сразу двум псевдоклассам и в другом виде может быть записан так: a:link, a:visited {…}.
a {color: #009}
a:any-link {color: #009}
IE не поддерживает данный элемент.
:target

Для перехода к выбранному фрагменту документа, в адресе пишется символ # и указывается имя идентификатора. К примеру, в адресе http://www.w3.org/TR/css3-selectors/#target-pseudo происходит переход к элементу, атрибут id которого задан как target-pseudo. Такая запись адреса называется целевой элемент.

Псевдокласс :target применяется к целевому элементу, иными словами, к идентификатору, который указан в адресной строке браузера.

Также этот псевдокласс стилизует «целевые элементы» при переходе внутри страницы по ссылкам (анкор).

:autofill

Позволяет стилизовать элементы input, когда браузер автоматически заполняет текстовое поле сохранёнными данными. Действие псевдокласса прекращается при редактировании поля формы. Возможность автозаполнение полей форм управляется через настройки браузера - например: «Автозаполнение и пароли», «Помнить историю поиска и данных форм».

:default

Псевдокласс :default применяет стиль к элементам форм, которые установлены по умолчанию в группе похожих элементов. Например, у нас может быть две кнопки для отправки данных формы на сервер. Одна из этих кнопок задана по умолчанию, мы можем изменить её стиль через :default.

form :default { ... }

<style>
  :default { background-color:lime; }
</style>

<form action="handler.php">
  <p>
    <input type="submit">
    <input type="submit">
    <input type="reset">
  </p>
</form>

В данном примере кнопка Submit, заданная по умолчанию, имеет другой цвет. Первая добавленная кнопка и является кнопкой по умолчанию.

IE не поддерживает данный элемент.

:disabled

используется для применения стиля к заблокированным элементам форм. Такие элементы не могут получить фокус, быть нажатыми или активированы, в текстовых полях нельзя набирать текст.

.elem:disabled {  }
<style type="text/css">
  textarea {background: yellow}
  textarea:disabled::placeholder {color: red}
  .ta4 {background: #fff}
</style>

<form>
  <textarea placeholder="введите текст"></textarea>
  <textarea class="ta4" placeholder="заблокированно" disabled></textarea>
</form>

:enabled

Псевдокласс :enabled используется для применения стиля к доступным (не заблокированным) элементам форм. По умолчанию, все элементы форм являются доступными, если в коде HTML к ним не добавляется атрибут disabled.

.elem:enabled { ... }

   else f.submit.disabled = 1;                  // берет из псевдомассива form
  }
</script>

:focus

Псевдокласс :focus определяет стиль для элемента, получающего фокус. Например, им может быть текстовое поле формы, в которое устанавливается курсор.

input:focus {               
  background:#ffc;
  outline:none;
  border-color:#a0d;
}

Псевдокласс :hover в сочетании со свойством transition часто применяется для создания различных анимационных эффектов при наведении курсора мыши на элемент. На мобильных устройствах :hover по понятным причинам не работает — там нет курсора, а взаимодействие происходит касанием пальца. Поэтому для создания эффектов заменим :hover на :focus.

Фокус могут получить не все элементы веб-страницы, а только ссылки, кнопки, поля формы. Чтобы заставить работать :focus для, скажем, изображений (img) или ссылки (a), надо добавить к ним атрибут tabindex — он позволяет переходить к изображениям через клавишу Tab. В качестве значения указываются целые числа, переход происходит от элемента с меньшим значением к элементу с большим значением.

<img src="image/thumb1.jpg" tabindex="1">

<a href="1.html" tabindex="2">Ссылка</a>

.elem:focus-visible {...}

:focus-within

определяет стиль элемента, когда он сам или элементы внутри него получают фокус. В отличие от :focus, который применяется непосредственно к самому элементу, :focus-within работает для родителя. Это позволяет выделять цветом или другими методами всю форму или отдельные её части, когда пользователь работает с полями формы.

:in-range

Применяется к элементам форм, у которых введённое пользователем значение находится в заранее заданном диапазоне. Сам диапазон устанавливается с помощью атрибутов min и max, они, соответственно, задают минимальное и максимальное значение.

Этот псевдокласс применяется только к элементам с заданным диапазоном ограничений. При отсутствии такого ограничения, элемент не может быть ни "в зоне допустимых значений", ни "вне диапазона".

Введите число от 1 до 10

<style>
  .form input:in-range {background: #eec;}
  .form input:out-of-range {background: #fcd;}
</style>

<form class="form">
  <p>Введите число от 1 до 10</p>
  <p><input type="number" min="1" max="10" value="1"></p>
</form>
IE не поддерживает данный элемент

:out-of-range

Псевдокласс применяется к полям форм, у которых введённое пользователем значение выходит из заданного диапазона. Псевдокласс работает только для тех полей, где пользователь может сам ввести значение, даже несмотря на ограничения наложенные атрибутами min и max.

Этот псевдокласс применяется только к элементам с заданным диапазоном ограничений. При отсутствии такого ограничения, элемент не может быть ни "в зоне допустимых значений", ни "вне диапазона".

Введите число от 1 до 10

:indeterminate

Псевдокласс :indeterminate задаёт стиль для переключателей, когда они находятся в неопределённом состоянии. К примеру, если ни один переключатель не помечен, то они находятся в указанном состоянии. В реальности, стиль применяется только к элементам, у которых DOM-атрибут :indeterminate через JavaScript установлен в значение true. Также этот псевдокласс используется некоторыми браузерами для элемента progress.

input:indeterminate { color: blue }
Селектор находит следующие элементы:
  • Элементы input type="checkbox", свойство indeterminate которых было установлено в true через JavaScript.
  • Элементы input type="radio", когда все радио переключатели в одной группе (с одинаковым атрибутом name) не выбраны.
  • Элементы progress в неопределённом состоянии.

:invalid

Псевдокласс :invalid находит любые input или form элементы, контент которых не проходит валидацию, в соответствии с типом поля. Он позволяет вам легко менять внешний вид полей, что позволяет пользователю видеть и исправлять ошибки.

Применяется к полям формы, содержимое которых не соответствует указанному типу. Например, для type="number" должно вводиться число, а не буквы, для type="email" корректный адрес электронной почты.

Псевдоклассы типа child и type.

:first-child :last-child :first-of-type :last-of-type :nth-child :nth-last-child :nth-last-of-type :nth-child(A of B) :nth-of-type :only-child :only-of-type

:first-child

Псевдокласс :first-child устанавливает стилевое оформление первого потомка элемента своего родителя.

:first-child { color: red }    /* каждый первый элемент где бы он ни был */
p:first-child { color: red }   /* каждый первый <p> где бы он ни был */
p :first-child { color: red }  /* каждый первый элемент внутри всех <p> */

При добавлении селектора к :first-child сперва берётся первый элемент, затем смотрится какого он типа. Если элемент совпадает с указанным селектором, то он будет выбран. Таким образом, выбирается элемент, если он удовлетворяет двум условиям одновременно: это первый элемент и это элемент указанного типа.

Для выбора первого элемента определённого типа используют псевдокласс :first-of-type.
Вместо :first-child допустимо использовать :nth-of-child(1).

:last-child

Псевдокласс :last-child позволяет настраивать характеристики оформления последнего элемента внутри родительского контейнера.

:last-child {...}

При добавлении селектора к :last-child сперва берётся последний элемент, затем смотрится какого он типа. Если элемент совпадает с указанным селектором, то он будет выбран. Таким образом, выбирается элемент, если он удовлетворяет двум условиям одновременно: это последний элемент и это элемент указанного типа.

Псевдокласс :last-child можно использовать в частности для списков, например если элементу списка <li> задать свойство border-bottom: 3px solid black; то в результате под всеми словами или предложениями из которых состоит список появятся сплошная линия чёрного цвета толщина которой составляет три пикселя. С эстетической точки зрения последнюю подчёркивающую линию можно убрать, с помощью li:last-child задав свойство border-bottom со значением ноль:

li {border-bottom: 3px solid black;}

li:last-child {border-bottom: 0;}  /* параметры последнего LI */

li :last-child {color: red}        /* цвет последнего элемента внутри каждого LI */
Для выбора последнего элемента определённого типа используют псевдокласс :last-of-type.

:first-of-type

Псевдокласс :first-of-type находит первого потомка своего типа среди детей родителя. Псевдокласс :first-of-type без указания селектора выберет первые элементы каждого типа. При добавлении селектора к :first-of-type сперва выбираются указанные элементы, затем из них берётся первый элемент.

.table tr:first-of-type { color: red }   /* первая строка в таблице */
.table td:first-of-type { color: red }   /* первая ячейка (внутри каждой строки) */
Вместо :first-of-type допустимо использовать :nth-of-child(1).

:last-of-type

Псевдокласс :last-of-type находит последнего потомка с заданным тегом в списке детей родительского элемента. Псевдокласс :last-of-type без указания селектора выберет последние элементы каждого типа (h1, p, ...) и установит для них указанные параметры.

ul li:last-of-type {color: red}   /* параметры последнего LI внутри каждого UL */

li :last-of-type {color: yellow}  /* цвет последних элементов внутри каждого LI */

:nth-child()

Псевдокласс :nth-child() находит один или более элементов, основываясь на их позиции среди группы соседних элементов. Псевдокласс :nth-child() указывается с единственным аргументом, описывающим паттерн для выбирания элементов.

Принимаемые значения:
  • odd - все нечётные номера элементов.
  • even - все чётные номера элементов.
  • число - порядковый номер дочернего элемента относительно своего родителя. Нумерация начинается с 1, это будет первый элемент в списке. Отрицательные значения игнорируются.
  • (A*n ± B) - где A и B — целые числа, а n — счётчик, который автоматически принимает значение 0, 1, 2.... A и B могут быть отрицательными числами, в итоге некоторые результаты могут также получиться отрицательными или равными нулю. Однако на элементы оказывают влияние только положительные значения из-за того, что нумерация элементов начинается с 1. Это позволяет отбросить некоторые элементы.

Предварительная фильтрация с помощью синтаксиса of

Новое в CSS Selectors Level 4 - возможность опционально передавать список селекторов в :nth-child() и :nth-last-child().

:nth-child(An+B [of S]?)
:nth-last-child(An+B [of S]?)

Когда указано of S, логика An + B применяется только к тем элементам, которые соответствуют заданному списку селекторов S. Это означает, что можно предварительно отфильтровать дочерние элементы, прежде чем An + B сделает свое дело.

Например, :nth-child(2 of .qqq) выбирает второй элемент, имеющий класс qqq. Другими словами: из всех дочерних элементов с классом qqq выберет второй.

Следует учесть, что S - это список селекторов, что означает, что он принимает несколько селекторов, разделенных запятой. Например, :nth-child(4 of .qqq, .www) выбирает четвертый элемент, который является либо .qqq, либо .www из множества дочерних элементов.

Этот способ можно использовать при оформлении строк динамических таблиц - при раскрашивании каждой второй строки, когда, например, часть строк становятся скрытыми.

tr:nth-child ( even of :not([hidden]) ) {  /* исключает скрытые строки */
   background-color: lightgrey;
}

Псевдоклассы :empty и :blank

Псевдокласс :empty выбирает все пустые элементы - это такие элементы, внутри которых нет дочерних элементов и текста, включая пробелы и переносы. Пустыми будут считаться следующие элементы.

<style>div:empy{...}</style>

<div></div>
<div><!-- Пустой --></div>
Соответственно, к непустым относятся следующие элементы. 
<div> </div> <!-- Есть пробел -->
<div> </div> <!-- Есть символ &nbsp; -->
<div>Бла-бла</div> <!-- Есть текст -->
<div><span></span></div> <!-- Есть дочерний элемент -->

<div>
</div> <!-- Есть перенос текста -->

Псевдокласс :blank работает схожим образом - считает пустыми элементы, содержащие пробел или перенос строк.

Псевдокласс :blank не поддерживается браузерами.

Псевдокласс :checked

Псевдокласс :checked применяется к элементам списков - option с атрибутом selected, а так же к тегу input у которого атрибут type имеет значение checkbox или radio и когда они находятся в положение «включено» или имеют атрибут checked.

примеры стилизации
input[type="radio"]:checked { ... }     /* Стили для включенных переключателей */

input[type="checkbox"]:checked { ... }  /* Стили для помеченных флажков (чекбоксов) */

option:checked { ... }                  /* Стили для выбранных элементов списка */

Псевдокласс :defined

Псевдокласс :defined применяется к элементу, когда он определён.

К таким элементам относятся следующие:
  • стандартные HTML-элементы, такие как p, h1, header и др.;
  • зарегистрированные пользовательские элементы.
Для обратного выбора — элементов, которые не были определены, можно использовать селектор :not(:defined).
.elem:defined { ... }
IE не поддерживает данный элемент.

Псевдокласс :dir

Псевдокласс :dir применяет стилевые правила к элементам на основе направления текста.

.elem:dir(ltr | rtl) { ... }

Для указания направления текста применяется атрибут dir со значением ltr или rtl. Поэтому в некоторых случаях селекторы :dir(rtl) и [dir="rtl"] будут давать одинаковый результат. Однако использование псевдокласса более универсально, поскольку оно основывается на направлении текста, которое определяет браузер.

<div dir="rtl">
  <p>…</p>
  <p>…</p>
</div>

p[dir="rtl"] { color:#333 }  /* не сработает т.к. у элемента <p> не задан атрибут dir. */
p:dir(rtl)   { color:#333 }

Псевдокласс :fullscreen

Применяется к элементам, когда браузер находится в полноэкранном режиме. При этом скрывается заголовок окна, адресная строка, вкладки, меню и другие стандартные элементы и браузер отображается на весь экран. Для перехода из стандартного режима в полноэкранный и обратно обычно применяется клавиша F11.

Чтобы псевдокласс работал, предварительно для отдельных элементов следует разрешить полноэкранный режим с помощью скриптов.

<!DOCTYPE html><html><head><meta charset="utf-8"> 
  <script>
   function fullScreen() {
    let el = document.querySelector('#msg');

    if (el.webkitRequestFullscreen) el.webkitRequestFullscreen();// Chrome, Opera, Safari
    else if (el.mozRequestFullScreen) el.mozRequestFullScreen(); // Firefox
    else if (el.msRequestFullscreen) el.msRequestFullscreen();   // Internet Explorer, Edge
    else if (el.requestFullscreen) el.requestFullscreen();       // Стандарт
   }
  </script>
  <style>
   body { margin: 0; }
   #msg {padding:10px; background:#bc7062; color:#fff;}
   #msg:-ms-fullscreen { background: #6088a5; }
   #msg:-webkit-full-screen { background: #6088a5; }
   #msg:-moz-full-screen { background: #6088a5; }
  </style>
</head>
<body>
  <div id="msg">
   Элемент страницы отображается в полноэкранном режиме.
  </div>
  <p><button onclick="fullScreen()">Посмотреть на полном экране!</button></p>
</body></html>
  • Internet Explorer поддерживает псевдокласс :-ms-fullscreen
  • Chrome, Opera, Safari поддерживают псевдокласс :-webkit-full-screen
  • Firefox поддерживает псевдокласс :-moz-full-screen
Псевдоклассы для каждого браузера следует писать отдельно, а не перечислять их через запятую.

Псевдокласс :has

Даёт возможность задавать стили родителя или предшествующего элемента на основе его дочерних элементов. Псевдокласс :has применяет стилевые правила к элементу, если у него есть указанный потомок или родственный элемент. Псевдокласс :has берет соответствующий список селекторов и их комбинаций в качестве аргумента.

h1:has(+ p) { font-weight: 700 }  /* для тегов <h1>, за которыми идёт тег <p>*/
a:has(img, .btn) { color: #333 }  /* элемент <a> с тэгом <img> или .btn внутри; */
button:has(.icon) { padding:5px}  /* элемент <button>, имеющий внутри элемент .icon*/
.car:not(:has(img)){ color:red }  /* выберет элемент .car, внутри которого нет <img>*/
:is(h1, h2):has(+ p) {color:red}  /* выберет элементы <h1>, <h2>, за которыми идёт <p>*/
li:has(+ li:hover) { color:red }  /* выберет предыдущий пункт списка при наведении */
ul li:hover + li { color:red }    /* выберет следущий пункт списка при наведении */
ul li:hover + li +li{color:red}   /* выберет второй после текущего пункт списка */
Внутри :has не разрешается вставлять другой :has, а также псевдоэлементы, например (::before, ::after).

Псевдокласс :hover

Псевдокласс :hover срабатывает, когда пользователь наводит на элемент мышью, но не обязательно активирует его, иными словами кнопка мыши не нажата.

.item:hover { color: #f30 }

Этот стиль может переопределяться другими относящимися к ссылкам псевдоклассами, такими как :link, :visited и :active, чтобы избежать этого необходимо их указывать в определенном порядке, а именно:

a:link, a:visited, a:hover, a:active, a:focus {}

Благодаря данному псевдоклассу можно существенно преобразить сайт.

Псевдокласс :is()

Псевдокласс :is() позволяет писать комбинированные селекторы в более краткой форме. Это функция-псевдокласс принимает список селекторов как аргумент, и выбирает любой элемент, который может быть выбран одним из селекторов в этом списке. Это полезно при переписи огромных списков селекторов в более компактную форму.

.elem :is(td, th) {padding: 0.5rem}  /* аналогично  .elem td, .elem th {...} */
:is(ol, ul) > li {margin: 0.5rem}    /* стили для li внутри ol или ul */
.box :is(h2, .title) {color: #222}   /* стили для h2 или .title внутри .box */
Селекторы, содержащие разные браузерные префиксы (-moz-, -webkit- и пр.), нельзя группировать вместе.

Псевдокласс :lang()

Данный псевдокласс используется когда в документе содержатся абзацы текста на разных языках. Чтобы браузер различал их, элементу с текстом добавляется атрибут lang с кодом языка, например, lang="fr". В результате чего этот элемент может быть стилизован при помощи селектора. С помощью псевдокласса :lang() можно задавать определённые настройки, характерные для разных языков, например, вид кавычек в цитатах.

p:lang(fr) { color: yellow }

Псевдокласс :not()

Псевдокласс-функция, принимающая селектор в качестве аргумента и находит элементы, не соответствующие селектору, т.е. задаёт правила стилей для элементов, которые не содержат указанный селектор.

div:not(p) { color:#333 }

В качестве селектора могут указываться единичные псевдоклассы, теги, идентификаторы, классы и селекторы атрибутов. Нельзя использовать псевдоэлементы и составные селекторы.

Kонструкция :not( :not(...) ) запрещена.
input:not([ type="submit" ]) { ... }
div:not( .head ) { ... }
Opera Android не поддерживает Selector list argument.

Псевдоклассы safari для audio и video.

:buffering применяется к элементам audio и video, когда происходит буферизация медиа-ресурса. Элемент считается буферизованным, если он пытается загрузить необходимые данные, но их ещё недостаточно для начала или продолжения воспроизведения. При буферизации ресурса псевдокласс :playing будет также будет работать.

:muted применяется к элементам audio и video, когда у них отключен звук.

:paused применяется к элементам audio и video, когда их проигрывание ставится на паузу. Состояние паузы возникает, если пользователь явно приостановил проигрывание медиа-ресурса или когда ресурс загружен, но проигрывание ещё не начато.

:playing применяется к элементам audio и video, когда происходит проигрывание аудио или видео. Если проигрывание остановлено по независящим от пользователя причинам, например, при буферизации данных, то :playing всё равно будет работать.

:seeking применяется к элементам audio и video, когда в проигрывателе на шкале времени ищется определённая позиция. Шкала времени в аудио или видео-плеере показывает прогресс воспроизведения медиа-ресурса, а также позволяет перематывать к желаемому фрагменту.

:stalled применяется к элементам audio и video, когда проигрывание аудио или видео застопорилось. Такое состояние возникает, когда не удаётся получить данные от медиа-ресурса в течение некоторого времени, например, из-за неполадок в сети. Несмотря на то, что фактического проигрывания медиа-ресурса в такой момент не происходит, браузер считает, что проигрывание продолжается и псевдокласс :playing будет работать.

:volume-locked применяется к элементам audio и video, когда громкость звука «заблокирована» пользователем.

audio:buffering { outline: 2px solid red }
audio:muted { outline: 2px solid orange }
audio:paused { opacity: .7 }
audio:playing { background-color: green }
audio:seeking { border-color: orange }
audio:stalled { background-color: pink }
audio:volume-locked { color: yellow }
