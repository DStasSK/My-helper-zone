Псевдоклассы и псевдоэлементы.

  • Псевдоклассы для тегов <input> <form> и <textarea>

:active

Чтобы пользователь понимал, что его щелчок по элементу сработал, в момент щелчка меняют вид элемента. Для этого используется псевдокласс :active, который добавляется к селектору.

устанавливает стиль ссылки в момент нажатия на неё. 
a:active {color:#f30;}

:link

Благодаря псевдоклассу можно изменить стиль ссылки и сайта в целом установив для неё необходимый формат. Этот параметр определяет необходимое стилевое оформление ссылкам, которые ещё не посещались.

a:link {color:#336699;}

:visited

Устанавливает стиль ссылки по которой был осуществлён переход. При использовании данного параметра, происходит замена стилей в соответствии с выбранными параметрами CSS, которыми могут быть цвет, фон и другие характерные опции.

a:visited {color:#336699;}

:target

Для перехода к выбранному фрагменту документа, в адресе пишется символ # и указывается имя идентификатора. К примеру, в адресе http://www.w3.org/TR/css3-selectors/#target-pseudo происходит переход к элементу, атрибут id которого задан как target-pseudo. Такая запись адреса называется «целевой элемент».

Псевдокласс :target применяется к целевому элементу, иными словами, к идентификатору, который указан в адресной строке браузера.

Также этот псевдокласс стилизует «целевые элементы» при переходе внутри страницы по ссылкам (анкор).

:empty и :blank

Псевдокласс :empty выбирает все пустые элементы, это такие элементы, внутри которых нет дочерних элементов и текста, включая пробелы и переносы. Пустыми будут считаться следующие элементы.

<style>div:empy{...}</style>

<div></div>
<div><!-- Пустой --></div>
Соответственно, к непустым относятся следующие элементы. 
<div> </div> <!-- Есть пробел -->
<div> </div> <!-- Есть символ -->
<div>Бла-бла</div> <!-- Есть текст -->
<div><span></span></div> <!-- Есть дочерний элемент -->
<div>
</div> <!-- Есть перенос текста -->

Псевдокласс :blank работает схожим образом - считает пустыми элементы, содержащие пробел или перенос строк. Псевдокласс :blank не поддерживается браузерами.

:checked

Псевдокласс :checked применяется к элементам интерфейса, таким как переключатели (checkbox), флажки (radio) и пункты списка (option), когда они находятся в положение «включено». Переключение элементов в такое состояние происходит с помощью атрибута checked элемента <input> или пользователем.

Более подробно написано в разделе :checked

input[type="radio"]:checked { ... }     /* Стиль применяется только ко включенным переключателям */
input[type="checkbox"]:checked { ... }  /* Стиль применяется только к помеченным флажкам (чекбоксам) */
option:checked { ... }                  /* Стиль применяется только к выбранным элементам списка */

:indeterminate

Псевдокласс :checked пзадаёт стиль для переключателей, когда они находятся в неопределённом состоянии. К примеру, если ни один переключатель не помечен, то они находятся в указанном состоянии. В реальности, стиль применяется только к элементам, у которых DOM-атрибут :indeterminate через JavaScript установлен в значение true. Также этот псевдокласс используется некоторыми браузерами для элемента <progress>.

Селектор:indeterminate { ... }

:fullscreen

Применяется к элементам, когда браузер находится в полноэкранном режиме. При этом скрывается заголовок окна, адресная строка, вкладки, меню и другие стандартные элементы и браузер отображается на весь экран. Для перехода из стандартного режима в полноэкранный и обратно обычно применяется клавиша F11.

Чтобы псевдокласс работал, предварительно для отдельных элементов следует разрешить полноэкранный режим с помощью скриптов.

HTML 
<!DOCTYPE html>
<html>
 <head><meta charset="utf-8"><title>:fullscreen</title>
  <script>
   function fullScreen() {
    var el = document.getElementById('msg'); // Получаем элемент
    if (el.webkitRequestFullscreen) el.webkitRequestFullscreen();// Chrome, Opera, Safari
    else if (el.mozRequestFullScreen) el.mozRequestFullScreen(); // Firefox
    else if (el.msRequestFullscreen) el.msRequestFullscreen();   // Internet Explorer, Edge
    else if (el.requestFullscreen) el.requestFullscreen();       // Стандарт
   }
  </script>
  <style>
   body { margin: 0; }
   #msg {padding:10px; background:#bc7062; color:#fff;}
   #msg:-ms-fullscreen { background: #6088a5; }
   #msg:-webkit-full-screen { background: #6088a5; }
   #msg:-moz-full-screen { background: #6088a5; }
  </style>
 </head>
 <body>
  <div id="msg">
   Страница отображается в полноэкранном режиме.
  </div>
  <p><button onclick="fullScreen()">Посмотреть на полном экране!</button></p>
 </body>
</html>
Свернуть
  • Internet Explorer поддерживает псевдокласс :-ms-fullscreen
  • Chrome, Opera, Safari поддерживают псевдокласс :-webkit-full-screen
  • Firefox поддерживает псевдокласс :-moz-full-screen

Псевдоклассы для каждого браузера следует писать отдельно, а не перечислять их через запятую.

:hover

устанавливает стиль элемента при наведения на него курсора мыши, без нажатия.

селектор:hover {color:#f30;}

Благодаря данному псевдоклассу можно существенно преобразить сайт.

:lang()

Данный псевдокласс используется когда в документе содержатся абзацы текста на разных языках. Чтобы браузер различал их, элементу с текстом добавляется атрибут lang с кодом языка, например, lang="fr". В результате чего этот элемент может быть стилизован при помощи селектора.

p:lang(fr) {css-стили}

:not

задаёт правила стилей для элементов, которые не содержат указанный селектор.

Селектор:not(<Селектор>) { ... }

В качестве селектора могут указываться единичные псевдоклассы, теги, идентификаторы, классы и селекторы атрибутов. Нельзя использовать псевдоэлементы и составные селекторы
:not (конструкция :not(:not(...)) запрещена).

input:not([type="submit"]) { ... }
div:not(.head) { ... }

:root

Более подробно написано в разделе :root

:first-child

Селектор или псевдокласс :first-child устанавливает стилевое оформление первого потомка элемента своего родителя.

:first-child {...}

:last-child

Селектор или псевдокласс :last-child позволяет настраивать характеристики оформления последнего элемента из ряда блоков содержащихся внутри родительского контейнера.

:last-child {...}

Псевдокласс :last-child можно использовать в частности для списков, например если элементу списка <li> задать свойство border-bottom: 3px solid black; то в результате под всеми словами или предложениями из которых состоит список появятся сплошная линия чёрного цвета толщина которой составляет три пикселя. С эстетической точки зрения последнюю подчёркивающую линию можно убрать, с помощью li:last-child задав свойство border-bottom со значением ноль:

li {border-bottom: 3px solid black;}

li:last-child {border-bottom:0;}
:nth-child() - элементы на основе их индекса (в порядке очереди)
:nth-child(-n+3) - выбрать только первые три элемента (3, 2, 1)
:nth-child(n+2):nth-child(-n+4) - выбрать (2, 3, 4)
:nth-child(even) - каждый четный элемент (2, 4, 6, 8, и т.д.)
:nth-child(odd) - каждый нечетный элемент (1, 3, 5, 7, и т.д.)
Использование :nth-child(2) без указания селектора выберет второй элемент любого типа

:nth-of-type() - элементы одного типа на основе их индекса внутри контейнера
img:nth-of-type(2n) {float: left;} - установит обтекание по левому краю каждой четной картинке, при условии, что каждая из них не обёрнута другим блоком.
Использование :nth-of-type(2) без указания селектора выберет второй элемент каждого типа.

Псевдоклассы :nth-child() и :nth-of-type() выбирают заданные элементы из группы братских элементов, начиная с первого. В противоположность им, псевдоклассы :nth-last-child() и :nth-last-of-type() выбирают элементы, считая от последнего.


:nth-last-child(2) - Выбирает предпоследний элемент.
:nth-last-child(-n+2) - Последние два элемента.
:nth-last-child(n+2) - Все элементы, кроме последних двух.

Псевдокласс :only-child выбирает дочерний элемент, только если он является единственным у своего родителя. Если у родителя два дочерних элемента, то они не выбираются. Например: p:only-child находит элемент только при соблюдении двух условий: это единственный дочерний элемент и это элемент <p>.

Псевдокласс :only-of-type выбирает дочерний элемент определённого типа, только если он является единственным таикм у своего родителя, наличие других элеметнов, отличного от заданного - не влияют на выбор.

Таким образом, p:only-child находит только единственный элемент и только если это <p>, а p:only-of-type находит элемент <p> и смотрит, является ли он единственным.

Псевдоэлементы

::before и ::after

Данные псевдо элементы позволяют добавлять контент до или после блоков, которым задан данный элемент. Кроме того, элементы, такие как <input>, <textarea> или <img> не содержат в себе другие элементы, таким образом псевдо-элементы к ним не применяются.

По умолчанию ::after создаёт строчный элемент.
span::after{content:'\A'; white-space:pre;}
перенос строки через псевдоэлемент ::after

::backdrop

Псевдоэлемент отображается ниже самого верхнего элемента в стеке по оси Z, но выше всех остальных элементов на странице, если они имеются. Обычно ::backdrop применяется для затемнения страницы, чтобы акцентировать внимание на фотографии или диалоговом окне, которые выводятся поверх такого затемнения.

В настоящее время работает только с модальными окнами, созданными с помощью элемента <dialog>.

Селектор::backdrop { ... }

Применяется со скриптами. Пример:

HTML 
<style>
  .primer-dialog {display:inline-block; margin-left:10px;}
  .primer-dialog dialog {width:300px;}
  .primer-dialog dialog::backdrop {background: rgba(0,0,0,0.7);}
</style>

<div class="primer-dialog">
  <button id="openDialog">Вход на сайт</button>
  <dialog>
    <form id="authentication" action="handler.php">
      <p><input name="login" required autofocus placeholder="Логин"></p>
      <p><input type="password" name="pass" required placeholder="Пароль"></p>
      <p><button type="submit">Войти</button>
      <button id="closeDialog">Закрыть окно</button></p>
    </form>
  </dialog>
  <script>
    var dialog = document.querySelector('dialog');
    document.querySelector('#openDialog').onclick = function() {dialog.showModal();}
    document.querySelector('#closeDialog').onclick = function() {dialog.close();}
  </script>
</div>

::first-letter

определяет стиль первого символа в тексте элемента, к которому добавляется. К этому псевдоэлементу могут применяться только стилевые свойства, связанные со свойствами шрифта, полями, отступами, границами, цветом и фоном. Hе работает на элементах со значением display: inline;. Ставим display: block или display: inline-block;

p::first-letter { ... }

::first-line

задаёт стиль первой строки форматированного текста. Длина этой строки зависит от многих факторов, таких как используемый шрифт, размер окна браузера, ширина блока, языка и т. д. В правилах стиля допустимо использовать только свойства, относящиеся к шрифту, изменению цвета текста и фона.

p::first-line { ... }

первую строку
можно отделить с помощью тега <br> и использовать как заголовок.

::selectionПозваляет задать цвет выделения текста.

Выделение обычно применяется для копирования в буфер фрагмента текста, чтобы после этого вставить выделенный фрагмент в другое место. Например, в текстовый редактор для дальнейшей правки текста. Сам цвет и фон выделенного фрагмента определяется настройками операционной системы, но с помощью CSS эти параметры можно задать самостоятельно.

В правилах стилей допускается использовать следующие свойства: color, background, background-color, cursor, outline и text-shadow.

Firefox до версии 62 использует нестандартный псевдоэлемент ::-moz-selection.

<style>
  ::selection {
    background:#809778;
    color:#fff;
  }
</style>

Псевдоклассы для тегов <input> <form> и <textarea>

:default

Псевдокласс :default применяет стиль к элементам форм, которые установлены по умолчанию в группе похожих элементов. Например, у нас может быть две кнопки для отправки данных формы на сервер. Одна из этих кнопок задана по умолчанию, мы можем изменить её стиль через :default.

Селектор:default { ... }

<style>
  :default { background-color:lime; }
</style>

<form action="handler.php">
  <p>
    <input type="submit">
    <input type="submit">
    <input type="reset">
  </p>
</form>

В данном примере кнопка Submit, заданная по умолчанию, имеет другой цвет. Первая добавленная кнопка и является кнопкой по умолчанию.

:disabled

используется для применения стиля к заблокированным элементам форм. Такие элементы не могут получить фокус, быть нажатыми или активированы, в текстовых полях нельзя набирать текст.

Селектор:disabled { ... }
<style type="text/css">
  .ta3 {width:300px; height:30px; resize:none; background:#fcfcfc; display:inline-block;}
  .ta4 {background:#eee;}
  textarea:disabled::placeholder {color:red;}
</style>

<form>
  <textarea class="ta3" placeholder="введите текст"></textarea>
  <textarea class="ta3 ta4" placeholder="заблокированно" disabled></textarea>
</form>

:enabled

Псевдокласс :enabled используется для применения стиля к доступным (не заблокированным) элементам форм. По умолчанию, все элементы форм являются доступными, если в коде HTML к ним не добавляется атрибут disabled.

Селектор:enabled { ... }

:focus

Псевдокласс :focus определяет стиль для элемента, получающего фокус. Например, им может быть текстовое поле формы, в которое устанавливается курсор.

Фокус могут получить не все элементы веб-страницы, а только ссылки, кнопки, поля формы. Чтобы заставить работать :focus для, скажем, изображений (<img>) или ссылки (<a>), надо добавить к ним атрибут tabindex — он позволяет переходить к изображениям через клавишу Tab. В качестве значения указываются целые числа, переход происходит от элемента с меньшим значением к элементу с большим значением.

Селектор:focus { ... }

<img src="image/thumb1.jpg" tabindex="1">

<a href="1.html" tabindex="2">Ссылка</a>

:focus-within

определяет стиль элемента, когда он сам или элементы внутри него получают фокус. В отличие от :focus, который применяется непосредственно к самому элементу, :focus-within работает для родителя. Это позволяет выделять цветом или другими методами всю форму или отдельные её части, когда пользователь работает с полями формы.

Введите ваше имя: 

<style>
  .f3 div {padding:10px;}
  .f3 div:focus-within {background:#DEF0E7;}
</style>

<form action="/example/handler.php" class="f3">
  <div>Введите ваше имя: <input name="user"></div>
  <p><button>Отправить</button></p>
</form>

:in-range

Применяется к элементам форм, у которых введённое пользователем значение находится в заранее заданном диапазоне. Сам диапазон устанавливается с помощью атрибутов min и max, они, соответственно, задают минимальное и максимальное значение.

Введите число от 1 до 10

<style>
  .f4 input:in-range {background: #e2edc1;}
  .f4 input:out-of-range {background: #f9c3d2;}
</style>

<form class="f4">
  <p>Введите число от 1 до 10</p>
  <p><input type="number" min="1" max="10" value="1"></p>
</form>

:out-of-range

Псевдокласс применяется к полям форм, у которых введённое пользователем значение выходит из заданного диапазона. Псевдокласс работает только для тех полей, где пользователь может сам ввести значение, даже несмотря на ограничения наложенные атрибутами min и max.

Введите число от 1 до 10

<style>
  .f6 input:in-range {background:#e2edc1;}
  .f6 input:out-of-range {background:#f9c3d2;}
  .f6 label {height:1em;}
  .f6 input:out-of-range + label::after {content:'Неверное число'; color:#ec008c; margin-left:0.5em;}
</style>

<form class="f6">
  <p>Введите число от 1 до 10</p>
  <p><input type="number" id="dec" min="1" max="10" value="1">
  <label for="dec"></label></p>
</form>

:invalid

Применяется к полям формы, содержимое которых не соответствует указанному типу. Например, для type="number" должно вводиться число, а не буквы, для type="email" корректный адрес электронной почты.

Адрес сайта

Адрес электронной почты

<style>
  .f5 input:invalid {background: #fdd;} /* Красный цвет фона */
  .f5 input:valid {background: #dfd;}   /* Зелёный цвет фона */
  .f50 {background:#ccc;}
  #f51 {background:#ccc;}
</style>

<form class="f5">
  <p>Адрес сайта</p>
  <p><input type="url" required></p>
  <p>Адрес электронной почты</p>
  <p><input type="email" required></p>
  <p><input type="submit" value="Отправить" id="f51"></p>
  <p><input type="submit" value="Отправить" class="f50"></p>
</form>

:valid

Применяется к полям формы, содержимое которых проходит проверку в браузере на соответствие указанному типу. Например, для type="number" вводится число, а не буквы, для type="email" корректный адрес электронной почты.

:optional

Псевдокласс применяет стилевые правила к элементу <input>, у которого не задан атрибут required (он позволяет выделять поля обязательные к заполнению перед отправкой формы). Таким образом :optional применяется к необязательным полям формы.

input:optional { ... }

:required

Псевдокласс применяет стилевые правила к элементу <input>, у которого установлен атрибут required, он позволяет выделять поля обязательные к заполнению перед отправкой формы.

input:required {  ...  }

::placeholder

Псевдоэлемент, с помощью которого задаётся стилевое оформление подсказывающего текста, созданного атрибутом placeholder. Допускается использовать свойства для изменения вида текста, например, задать шрифт и цвет.

<style>
  input[type="search"]::-moz-placeholder          {color:#009}
  input[type="search"]:-moz-placeholder          {color:#009}
  input[type="search"]::-webkit-input-placeholder {color:#009}
  input[type="search"]:-ms-input-placeholder      {color:#009} /* не ошибка */
  input[type="search"]::-ms-input-placeholder     {color:#009} /* их два */
  input[type="search"]::placeholder               {color:#009}
</style>
Т.к. в разных браузерах цвет шрифта placeholder может быть прозрачной, то лучше дополнительно указывать параметр opacity 
  ::-webkit-input-placeholder {color:#f00; opacity:1;}
  ::-moz-placeholder          {color:#f00; opacity:1;}
  :-moz-placeholder           {color:#f00; opacity:1;}
  :-ms-input-placeholder      {color:#f00; opacity:1;}
IE8 не поддерживают атрибут «placeholder» — он просто не будет отображаться. Для отображени подсказки в браузерах, которые не поддерживают атрибут, необходимо использовать скрипты.

:placeholder-shown

Определяет стиль элемента <input> или <textarea>, который в данный момент отображает текст подсказки, заданной атрибутом placeholder (при этом текст в поле еще не введен).

Так же можно прятать текст подсказки при фокусе поля ввода - задавая ей цвет как transparent;

input:focus:placeholder-shown {-webkit-text-fill-color:transparent;}

:read-only

Псевдокласс применяется к полям формы, у которых задан атрибут readonly. Такое поле не может быть модифицировано, при этом оно получает фокус, а данные в таком поле можно выделить и скопировать.

Firefox поддерживает псевдокласс :-moz-read-only.

input:-moz-read-only { ... }
input:read-only { ... }

<style>
  #f7 input:read-only {opacity:0.5;} /* Полупрозрачность */
</style>

<form action="handler.php" id="f7">
  <p><input type="text" name="comment" size="40"
  value="Введенный текст не может изменяться" readonly></p>
  <p><input type="submit" value="Отправить"></p>
</form>

:read-write

Псевдокласс применяется к полям формы, доступных для изменения. Псевдокласс :read-write является противоположным по своему действию :read-only, который применяется к полям с атрибутом readonly (только для чтения).

Firefox поддерживает псевдокласс :-moz-read-write.

input:-moz-read-write { ... }
input:read-write { ... }

Как видно из примера элемнт <input> с параметром disabled применяет на себя параметры заданные псевдоклассом :read-write.

<style>
  #f8 input:read-only {border: 3px solid grey;}
  #f8 input:read-write {border: 3px solid green;}
</style>

  <p><input type="text" size="40" value="disabled" disabled></p>
