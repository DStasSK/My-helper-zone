Элемент <script> (от англ. script — сценарий) предназначен для описания скриптов, может содержать ссылку на программу или её текст на определённом языке. Скрипты могут располагаться во внешнем файле и связываться с любым HTML-документом. Такой подход позволяет использовать одни и те же общие функции на многих веб-страницах и ускоряет их загрузку, т. к. внешний файл кэшируется при первой загрузке, и скрипт вызывается быстрее при последующих вызовах.

<script> может располагаться в заголовке или теле HTML-документа в неограниченном количестве. В большинстве случаев местоположение скрипта никак не сказывается на работу программы. Однако скрипты, которые должны выполняться в первую очередь, обычно помещают в заголовок документа.

Для правильной работы скриптов их обычно размещают в конце документа. Так как для работы скрипта зачастую требуются элементы сраницы, но они еще не будут загружены.
<script>
  код скрипта
</script>

<script src="адрес"></script>

Атрибуты

async Загружает скрипт асинхронно. Это означает, что указанный в атрибуте src файл будет выполняться без ожидания загрузки и отображения веб-страницы. В то же время и страница не ожидает результата выполнения скрипта, а продолжает загружаться как обычно. По умолчанию этот атрибут выключен. 
<script async>...</script>
Если задано сразу два атрибута - async и defer, то атрибут async игнорируется.
defer

Откладывает выполнение скрипта до тех пор, пока вся страница не будет загружена полностью. Работает только при наличии атрибута src. По умолчанию этот атрибут выключен.

Этот атрибут обеспечивает возможность отложенного исполнения скрипта, сообщая браузеру о том, что данный сценарий не производит генерирования содержимого Web-страницы. Если этот атрибут не установлен, то отображение страницы пользовательским агентом прерывается до момента завершения исполнения всех инструкций, определяемых в элементе SCRIPT.

<script src="адрес" defer></script>
language Устанавливает язык программирования на котором написан скрипт. 
<script
  language="JavaScript | JScript | VBS или VBScript">
</script>

<script
  type="text/vbscript" language="VBScript">
</script>
<script
  type="text/javascript" language="JavaScript">
</script>
Данный атрибут осуждается, вместо него следует применять type, который указывает MIME-тип для определённого языка.
src Адрес скрипта из внешнего файла для импорта в текущий документ. Как только внешние инструкции полностью загружаются, браузер обрабатывает их, словно они были внедрены в текущий документ. В качестве значения принимается полный или относительный путь к файлу. Файл должен иметь расширение .js.
При указании ссылки на файл со скриптом и наличии скрипта в теле самого тега script - скрипт внутри тега игнорируется. Т.е. выполняется только скрипт из документа по ссылке.
type Определяет тип содержимого <script>.
text/javascript - для языка программирования JavaScript.
text/vbscript - для языка VBScript. 
<script type="text/javascript"></script>
В HTML5 атрибут type можно опустить, он является необязательным и принимает значение text/javascript, если не указан явно. В предыдущих версиях HTML атрибут type необходим.

Включение кода в виде псевдо-URL

Данный способ подразумевает формирование URL специального вида путем указания протокола доступа к данным javascript, после чего записывается программный код в виде одной строки. Инструкции в строке должны быть отделеный друг от друга точкой с запятой ( символ ; ), а комментарии располагаются внутри комбинации символов /* */ (что является заменой //).

"Ресурсом", который определяется URL-адресом, является значение, возвращаемое тим програмным кодом, преобразованного в строку, если код возвращает undefined, то считается что ресурс не имеет содержимого.

Подобный URL, например, может быть задан в виде:
  • значения атрибута href тега a
  • значения атрибута action тега form
  • значения атрибута src тега form илиimg
  • агрумента метода window.open()

Некоторые браузеры после выполнения кода в виде URL, используют возвращаемое значение как содержимое нового документа, как при переходе по ссылке; при этом содержимое текущего документа затирается.

Если необходимо гарантировать, что после выполнения кода внутри URL-инструкции, текущий документ не затрется, то можно использовать оператор void, который после выполнения кода возвращает undefined.

<a href="javascript:void window.open('about:blank');">Открыть окно</a>
При использовании данного метода включения JavaScript-кода в HTML-документ обязательно указание для документа языка сценариев по умолчанию.

Это может быть сделано путем включения соответствующего объявления META-информации в тег head: либо HTTP-заголовком Content-Script-Type:text/javascript.

<META http-equiv="Content-Script-Type" content="text/javascript">

Данный способ часто используют с целью создания на странице ссылок для добавления этой страницы в список закладок браузера и других похожих задач.

Пример 1.

В данном примере с помощью скрипта выводится таблица, состоящая из пяти строк и столбцов, которая заполняется числами.

<div id="forTable"></div>

<script>
  let table = document.querySelector('#forTable');
  let codeHtml = '<table>';

  for (i = 1; i < 6; i++) {
    codeHtml += '<tr>';
    for (j = 1; j < 6; j++) {
      codeHtml += ('<td>' + i + j + '</td>');
    }
    codeHtml += '</tr>'
  }

  codeHtml += '</table>';
  table.innerHTML = codeHtml;
</script>

Пример 2.

В данном примере внутри элемента с идентификатором welcome выводится текст сообщения.

Тескт, который исчезнет по клику
<div id="welcome">
  Тескт, который исчезнет по клику
</div>

<script>
  let el = document.querySelector('#welcome');
  el.onclick = popup;

  function popup() {
    el.innerHTML = 'Добро пожаловать!';
  }
</script>
