Элемент link (от англ. link — ссылка) - определяет отношения между текущим документом и внешним ресурсом. Этот элемент чаще всего используется для ссылки на таблицы стилей и для установки иконок сайта (как иконок в стиле "favicon", так и иконок для домашнего экрана и приложений на мобильных устройствах). Обычно размещается внутри контейнера head.

<head>
  <link rel="stylesheet" href="url">
  <link rel="icon" href="cite.ico">
</head>

Атрибуты.

as disabled blocking charset crossorigin fetchpriority href hreflang imagesizes imagesrcset integrity media referrerpolicy rel rev sizes target title type
as

Указывает тип контента, загружаемого link, что необходимо для сопоставления запросов, применения правильной политики безопасности контента и установки правильного заголовка запроса Accept.

Этот атрибут используется только для элементов link с атрибутами rel="preload"; необязателен, если задано rel="modulepreload", в противном случае атрибут не должен использоваться.

<link rel="preload"
      as="audio | document | embed | fetch | font | image |
          object | script | style | track | video | worker">
disabled

Boolean атрибут указывающий, следует ли загружать и применять описанную таблицу стилей к документу.

Он должен использоваться только для элементов link с атрибутoм rel="stylesheet".

Если в HTML при загрузке указано disabled, таблица стилей не будет загружена во время загрузки страницы. Вместо этого таблица стилей будет загружена по запросу, если и когда атрибут disabled будет изменен на false или удален.

<link rel="stylesheet" href="url" disabled>
Установка свойства disabled в DOM приводит к удалению таблицы стилей из списка Document.styleSheets документа.
blocking

Этот атрибут явно указывает, что определенные операции должны быть заблокированы при извлечении внешнего ресурса.

Он должен использоваться только для элементов link с атрибутами rel="expect" или rel="stylesheet", в противном случае атрибут не должен использоваться.

<link rel="expect" blocking="render" href="url">
render - отображение контента на экране заблокировано.
charset
Данный атрибут устарел, не используйте его.
Указывает кодировку документа на который ссылается элемент link. 
<link charset="кодировка">
<link href="okinawa.html" rel="parent" charset="euc-jp">

<!-- сейчас используют -->
<meta charset="utf-8">
crossorigin Описывает, как элемент обрабатывает CORS-запросы, предназначен для использования со ссылками на внешние ресурсы. 
<link href="url" rel="stylesheet"
      crossorigin="anonymous | use-credentials">

fetchpriority Дает подсказку относительного приоритета для использования при извлечении ресурса определенного типа. 
<link href="url" fetchpriority="high | low | auto">
  • high - извлечь ресурс с высоким приоритетом относительно других ресурсов того же типа.
  • low - извлечь ресурс с низким приоритетом относительно других ресурсов того же типа.
  • auto - ие устанавливать предпочтение для приоритета извлечения. Это значение по умолчанию. Оно используется, если не задано значение или задано недопустимое значение.
href Путь к связываемому файлу - на который делается ссылка. 
<link href="адрес">
hreflang Задает язык связанного ресурса, добавляется только при наличии атрибута href. 
<link href="url" hreflang="язык">
imagesizes

Указывает размер иконок для визуального отображения. Добавляется только при наличии атрибутов rel="preload" и as="image".

<link rel="preload" as="image" imagesizes="values">
imagesizes имеет синтаксис аналогичный атрибуту sizes.
imagesrcset

Указывает на необходимость предварительной загрузки соответствующего ресурса, используемого элементом img, с соответствующими значениями для его атрибутов srcset и sizes.

Добавляется только при наличии атрибутов rel="preload" и as="image". 
<link rel="preload" as="image" href="img.jpg"
      imagesrcset="img_600.jpg 600w, img_1200.jpg 1200w"
      imagesizes="100vw">
imagesrcset имеет синтаксис аналогичный атрибуту srcset элемнта img.
integrity

Содержит хэш стилевого файла, расположенного на стороннем сервере. Браузер загружает стилевой файл, указанный в атрибуте href элемента link, создаёт хэш файла и сравнивает полученное значение со значением атрибута integrity. Если хэши различаются, то выполнение стилевого файла блокируется в целях безопасности.

Добавляется только при наличии атрибута rel со значениями stylesheet, preload или modulepreload

. 
<link rel="stylesheet" href="https://any.com/file.css"
      integrity="sha256-X7rrn44l1+AUO65h1LGbOstSYshT8=">
media

Определяет устройство, для которого следует применять стилевое оформление. Это позволяет сделать разный стиль для отображения документа на экране монитора и при его печати. Допускается писать несколько значений через запятую.

<link media="all | print | screen | speech">
  • all - Все устройства (значение по умолчанию).
  • print - Печатающее устройство вроде принтера.
  • screen - Экран монитора.
  • speech - Речевые синтезаторы, а также программы для воспроизведения текста вслух. Сюда же входят речевые браузеры.
HTML5 распространил это на любые медиавыражения, которые являются расширенным набором допустимых значений HTML4.
Браузеры, не поддерживающие медиавыражения, могут не распознать соответствующую ссылку.
пример.
referrerpolicy

Oпределяет политику, какие данные включать в заголовок Referer и отправлять при подключении внешнего документа в элементе link - никакие, ограниченные или все.

Заголовок Referer в HTTP-запросе содержит адрес страницы, с которой был загружен внешний документ. Например, если пользователь со страницы https://www.ru/a загрузил документ с другого сайта, то в запросе будет следующий заголовок: https://www.ru/a.

<link referrerpolicy="no-referrer |
                      no-referrer-when-downgrade |
                      origin |
                      origin-when-cross-origin |
                      unsafe-url |
                      same-origin |
                      strict-origin |
                      strict-origin-when-cross-origin">
  • no-referrer - заголовок Referer не отправляется.
  • no-referrer-when-downgrade - заголовок Referer не будет отправлен отправляется при переходе с защищённого протокола на обычный (HTTPS→HTTP). Значение по умолчанию.
  • origin - реферальные данные ограничены и включают только протокол, адрес сайта и порт, если он указан (https://example.com:80).
  • origin-when-cross-origin - в рамках одного сайта передаётся полный путь. При переходе на другой сайт реферальные данные ограничены и включают только протокол, адрес сайта и порт.
  • unsafe-url - реферальные данные будет включать источник и путь (но не фрагмент, пароль или имя пользователя). Это значение считается небезопасным.
  • same-origin - реферальные данные отправляются в пределах одного сайта, при переходе на другой сайт данные не передаются.
  • strict-origin - реферальные данные включают протокол, адрес сайта и порт (при его наличии). Сами данные передаются только в рамках одного и того же протокола (HTTPS→HTTPS) и не отправляются при переходе на менее безопасный (HTTPS→HTTP).
  • strict-origin-when-cross-origin - передаёт полный путь в рамках одного сайта, на другой сайт передаётся протокол, адрес сайта и порт, но только когда протокол остаётся прежним (HTTPS → HTTPS). Заголовок не передаётся при переходе на менее безопасный протокол (HTTPS→HTTP).
rel Определяет отношения между текущим документом и файлом, на который делается ссылка. Это необходимо, чтобы браузер знал, как использовать подключаемый документ. 
<link rel="тип_документа">
  • alternate - альтернативное представление текущего документа, используется, к примеру, для указания ссылки на файл в формате XML для описания ленты новостей, анонсов статей.
  • author - указывает, что указанный документ содержит дополнительную информацию об авторе текущего документа или статьи.
  • canonical - определяет предпочтительный URL для текущего документа, что помогает поисковым системам сократить дублирование контента.
  • help - Ключевое слово указывает, что связанный с содержимым элемент предоставляет контекстно-зависимую справку, предоставляя информацию для родителя элемента, определяющего гиперссылку, и его дочерних элементов.
  • icon - Связанный ресурс представляет значок, ресурс для представления страницы в пользовательском интерфейсе для текущего документа.
    Если есть несколько link rel="icon", браузер использует их атрибуты media, type и sizes для выбора наиболее подходящего значка и использует последний. Если позднее оказывается, что он не подходит, например, потому что он использует неподдерживаемый формат, браузер переходит к следующему наиболее подходящему и т. д.
    Примечание: iOS от Apple не использует этот тип ссылки, а также атрибут sizes, как это делают другие мобильные браузеры, для выбора значка веб-страницы для Web Clip или заполнителя запуска. Вместо этого он использует нестандартные apple-touch-icon и apple-touch-startup-image соответственно.
  • license - указывает, что гиперссылка ведет к документу, описывающему информацию о лицензировании; что основное содержимое текущего документа покрывается лицензией авторского права, описанной в указанном документе.
    <link rel="license" href="#license" />
  • manifest - Манифест веб-приложения. Требует использования протокола CORS для кросс-источниковой выборки.
    не поддерживается в FF
  • modulepreload - (2023.09) предоставляет декларативный способ упреждающего извлечения скрипта модуля, его анализа и компиляции, а также сохранения в карте модулей документа для последующего выполнения.
    Не рекомендуется загружать все скрипты на странице таким образом, это пожет привести к сбою браузера.
    Ссылки с rel="modulepreload" похожи на ссылки с rel="preload". Главное отличие в том, что preload просто загружает файл и сохраняет его в кэше, в то время как modulepreload получает модуль, анализирует и компилирует его и помещает результаты в карту модуля, чтобы он был готов к выполнению.
    Атрибут as является необязательным и по умолчанию имеет значение script, но он также может быть установлен на audioworklet, paintworklet, serviceworker, sharedworker или worker.
    <link rel="modulepreload" href="main.js">
    <link rel="modulepreload" href="modules/canvas.js">
    <link rel="modulepreload" href="modules/square.js">
  • next - Сообщает, что текущий документ является частью связанных между собой документов, а ссылка указывает на следующий документ.
  • preload - позволяет объявлять запросы, указывая ресурсы, которые вашей странице понадобятся очень скоро. Он не загружает и не выполняет скрипт, а только планирует его загрузку и кэширование с более высоким приоритетом. Также нужно будет указать путь к ресурсу в атрибуте href и тип ресурса в атрибуте as со значениями fetch, font, image, script, style. 
    <link rel="preload"
          href="600.css" as="style"media="(max-width:600px)">
    <link rel="preload"
          href="main.js" as="script">
    <link rel="preload"
          href="flower.avif" as="image" type="image/avif">
    <link rel="preload"                                   
          href="flower.webp" as="image" type="image/webp">
    Если браузер не поддерживает тип ресурса, то ресурс не будет загружатся.
    Указывать предварительную загрузку для нескольких типов одного и того же ресурса не рекомендуется, так как они будут загружены оба. Вместо этого лучше всего указывать предварительную загрузку только для того типа, который большинство ваших пользователей, скорее всего, будут использовать.
  • prev - Сообщает, что текущий документ является частью связанных между собой документов, а ссылка указывает на предыдущий документ.
  • privacy-policy - указывает, что указанный документ является Политикой конфиденциальности, которая описывает методы сбора и использования данных текущего документа.
  • search - указывают, что гиперссылка ссылается на документ, интерфейс которого специально разработан для поиска в текущем документе, на сайте и связанных ресурсах, предоставляя ссылку на ресурс, который можно использовать для поиска.
  • stylesheet - Определяет, что подключаемый файл хранит таблицу стилей (CSS).
  • terms-of-service - указывает, что указанный документ является Условиями обслуживания, описывающими соглашения между поставщиком текущего документа и пользователями, желающими использовать предоставленный документ.
  • apple-touch-icon - Задает значок веб-приложения на устройстве iOS.
  • apple-touch-startup-image - изображение экрана запуска на устройстве iOS, которое будет отображаться при запуске веб-приложения. Это полезно, когда веб-приложение находится в автономном режиме. По умолчанию используется снимок экрана веб-приложения при последнем запуске. 
    <link rel="apple-touch-startup-image" href="/launch.png">
rev
устарело и не поддерживается
Описывает обратное отношение текущего документа к связываемому документу, как определено атрибутом href.
sizes Этот атрибут определяет размеры иконки для визуальных медиа, содержащихся в ресурсе. Используется только, если атрибут rel содержит значение icon или apple-touch-icon. 
<link rel="icon" sizes="48x48" 
      href="main.ico" type="image/x-icon">
<link rel="icon" sizes="16x16" 
      href="icon.png" type="image/png">
Вначале указывается ширина иконки в пикселах без указания единиц, затем пишется латинская буква x в верхнем или нижнем регистре, после чего идёт высота иконки. Если в файле хранится сразу несколько иконок, можно задавать их размеры через пробел. Ключевое слово any указывает, что иконка может масштабироваться в любой размер, к примеру, если она хранится в векторном формате SVG. 
<link rel="icon" href="main.ico" type="image/x-icon"
      sizes="ширина1Xвысота1 ширина2Xвысота2 | any">
title
Этот атрибут поддерживается только FF.
Атрибут title имеет особую семантику для элемента link. При использовании rel="stylesheet" он определяет таблицу стилей по умолчанию или альтернативную таблицу стилей. Неверное использование может стать причиной игнорирования таблицы стилей.
type Используется для определения типа контента, на который ссылаются. Значение атрибута должно быть MIME типом, например text/html, text/css и т.д. Значение по умолчанию - text/css. Данный атрибут также используется для типов ссылок rel="preload", чтобы браузер загружал только поддерживаемые им типы файлов. 
<link rel="stylesheet" href="css/main.css" type="text/css">
Помимо собственных атрибутов, в теге могут быть использованы глобальные атрибуты, доступные для всех тегов HTML документа.

Пример подключения каскадных таблиц стиля.

В следующем примере подключается внешний файл со стилями с помощью атрибута rel="stylesheet" элемента link, указывается RSS-документ текущего сайта (rel="alternate" type="application/rss+xml") и устанавливается иконка сайта в адресной строке браузера (link rel="icon").

<head>
  <link rel="icon" href="favicon.ico">
  <link rel="shortcut icon" href="/img/logo.png" type="image/png"> HTML4
  <link rel="shortcut icon" href="favicon.ico" type="image/x-icon"> HTML4
  <link rel="stylesheet" type="text/css" href="main.css">
  <link rel='stylesheet' type='text/css'
        href='https://fonts.googleapis.com/css?family=Open+Sans:300italic,
              600&subset=latin,cyrillic'>
  <link rel="alternate" type="application/rss+xml" 
        title="Статьи с сайта www.ru" href="https://www.ru/rss.xml">
</head>
