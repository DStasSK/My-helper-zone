CSS cвойство display

Свойство display устанавливает: будет ли элемент рассматриваться как блок или встроенный блок; это свойство также может определять макет, используемый для размещения его дочерних элементов - flow layout, grid или flex. Формально свойство display устанавливает внутренние и внешние типы отображения элемента. Внешний тип устанавливает участие элемента в потоковом макете; внутренний тип устанавливает макет дочерних элементов.

div {
/* частоиспользуемые значения */
  display: block | inline | inline-block |flex | inline-flex |
           grid | inline-grid | contents | list-item | none |
           flow | flow-root;

/* значения для элементов таблиц */
  display: table | inline-table | table-caption | table-cell | table-row |
           table-row-group | table-column | table-column-group |
           table-header-group | table-footer-group

/* Global values */
  display: inherit | initial | revert | revert-layer | unset;

/* multi-keyword syntax - для самых современных браузеров */
  display: block flex | block flow | block flow-root | block grid |
           inline flex | inline flow | inline flow-root | inline grid;
}
block Элемент показывается как блочный - генерирует разрывы строк как до, так и после элемента в нормальном потоке. Применение этого значения для встроенных элементов, например тега span, заставляет его вести подобно блокам - происходит перенос строк в начале и в конце содержимого.
inline

Элемент отображается как встроенный - генерирует один или несколько встроенных блоков, которые не генерируют разрывы строк до или после себя. В нормальном потоке следующий элемент будет на той же строке, если есть место.

Использование блочных тегов, таких как div и p, автоматически создает перенос и показывает содержимое этих тегов с новой строки. Значение inline отменяет эту особенность, поэтому содержимое блочных элементов начинается с того места, где окончился предыдущий элемент.
Когда браузеры, поддерживающие синтаксис с несколькими ключевыми словами, сталкиваются со свойством display, имеющим только внешнее значение (например, display: block или display: inline), внутреннее значение устанавливается равным flow (например, display: block flow и display: inline flow).
flow

Элемент размещает свое содержимое с помощью макета потока: eсли его внешний тип отображения — inline, и он участвует в блочном или инлайновом контексте форматирования, то он генерирует инлайновый блок. В противном случае он генерирует блочный блок.

В зависимости от значения других свойств (таких как position, float или overflow) и того, участвует ли он сам в блочном или инлайновом контексте форматирования, то он либо устанавливает новый блочный контекст форматирования для своего содержимого, либо интегрирует свое содержимое в свой родительский контекст форматирования.
flow-root Элемент генерирует блочный блок, который устанавливает новый блочный контекст форматирования, определяя, где находится корень форматирования.
inline-block Это значение генерирует блочный элемент, который обтекается другими элементами веб-страницы подобно встроенному элементу. Фактически такой элемент по своему действию похож на встраиваемые элементы (вроде img). При этом его внутренняя часть форматируется как блочный элемент, а сам элемент - как встроенный.
Эквивалентно значению inline flow-root
contents Заставляет контейнер исчезнуть, делая дочерние элементы дочерними элементами элемента следующего уровня в DOM. Элемент не создаёт определенный блок сам по себе. Он заменяется своими псевдоблоками и дочерними блоками.
none

Отключает отображение элемента, чтобы он не влиял на макет (документ отображается так, как будто элемент не существует). У всех дочерних элементов также отключено отображение. Изменить значение и сделать элемент вновь видимым можно с помощью скриптов или CSS селекторов.

Чтобы элемент занимал место, которое он обычно занимает, но фактически ничего не отображал, используют свойство visibility.
flex Элемент ведёт себя как блочный и выкладывает содержимое согласно флекс-модели.
inline-flex Элемент ведет себя как элемент встроенного уровня и размещает свое содержимое в соответствии с моделью flexbox.
grid Отображает элемент в виде контейнера сетки на уровне блоков.
inline-grid Элемент ведет себя как элемент встроенного уровня и размещает свое содержимое в соответствии с моделью сетки.
list-item Элемент выводится как блочный и добавляется маркер списка, т.е. элемент ведет себя как элемент li.

Значения для элементов таблиц.

table Определяет, что элемент является блочной таблицей подобно использованию тега table.
inline-table Определяет, что элемент является таблицей как при использовании тега table, но при этом таблица является встроенным элементом и происходит ее обтекание другими элементами, например, текстом.
table-caption Задает заголовок таблицы подобно применению тега caption.
table-cell Указывает, что элемент представляет собой ячейку таблицы (тег <td> или <th>).
table-row Элемент отображается как строка таблицы (тег <tr>).
table-column Назначает элемент колонкой таблицы, словно был добавлен тег <col>.
table-column-group Определяет, что элемент является группой одной или более колонок таблицы, как при использовании тега <colgroup>.
table-footer-group Используется для хранения одной или нескольких строк ячеек, которые отображаются в самом низу таблицы. По своему действию сходно с работой тега <tfoot>.
table-header-group Элемент предназначен для хранения одной или нескольких строк ячеек, которые представлены вверху таблицы. По своему действию сходно с работой тега <thead>.
table-row-group Создает структурный блок, состоящий из нескольких строк таблицы аналогично действию тега <tbody>.

Следующие параметры поддерживают не все браузеры

ruby Элемент ведет себя как элемент встроенного уровня и размещает свое содержимое в соответствии с моделью форматирования ruby. Он ведет себя как элемент HTML ruby.
ruby-base Эти элементы ведут себя как элементы HTML rb.
ruby-text Эти элементы ведут себя как элементы HTML rt.
ruby-base-container Эти элементы генерируются как анонимные блоки.
ruby-text-container Эти элементы ведут себя как элементы HTML rtc.
run-in Устанавливает элемент как блочный или встроенный в зависимости от контекста.

multi-keyword syntax

Cвойство display имеет значительные возможности. Помимо указания того, является ли что-то блоком уровня блока или встроенным по отношению к другим элементам на странице, оно также указывает контекст форматирования внутри элемента, к которому оно применяется. Чтобы лучше описать это поведение, свойству display устанавливают два значения — внешнее и внутреннее. Исходный синтаксис с одним значением также допустим. Внешний тип отображения описывает, является ли элемент блочным или строчным. Внутренний тип отображения описывает, как ведут себя дочерние элементы этого блока.

Например, когда используют display: flex, создают контейнер блочного уровня с flex-дочерними элементами. Дочерние элементы описываются как участвующие в контексте форматирования flex. Вместо установки display: flex для создания блока с дочерними элементами flex используют display: block flex.
