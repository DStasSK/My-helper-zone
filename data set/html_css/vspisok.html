Ваиант на тегах <details> и <summary>.

Архитектура:

<details>
  <summary>Видимый текст 1</summary>
  <p>Скрытый текст 1</p>
  <p>Скрытый текст 1</p>
</details>
<details>
  <summary>Видимый текст 2</summary>
  <p>Скрытый текст 2</p>
  <p>Скрытый текст 2</p>
</details>

Дефолтное отображение:

Видимый текст 1

Скрытый текст 1

Скрытый текст 1

Видимый текст 2

Скрытый текст 2

Скрытый текст 2

Минусы данного метода:
- старые браузеры не знают таких тегов и не будут скрывать информацию.
- дефолтный маркер надо спрятать, используя префиксы, и создать свой.

Чтобы тег <summary> не растягивался на всю ширину и были кликабельными только слова, а не вся строка - устанавливают параметр display:inline-block.

summary::-webkit-details-marker {display: none;}
summary::-moz-list-bullet {list-style-type: none;}
summary::marker {display: none;}
summary {display: inline-block;}
IE не поддерживает данные тэги.

Вариант на onclick и toggle.

Нажми меня два раза
  • 1-й пункт
  • 2-й пункт
  • 3-й пункт
HTML 
<!-- id повторяться не должны -->
<div class="menu" onclick='submenu.classList.toggle("hiden");'>
  Нажми меня два раза
</div>

<!-- event.stop... не позволят кликам по пунктам сворачивать меню
т.е. предотвращает всплытие клика
onclick="event.stopPropagation()" -->
<ul id='submenu' class="hiden" onclick="event.stopPropagation()">
  <li>1-й пункт</li>
  <li>2-й пункт</li>
  <li>3-й пункт</li>
</ul>

CSS 
.menu {cursor:pointer; display:inline-block;}
.hiden {display:none;}

Вариант на <input>, <label> и :checked.

Есть два типа аккордеон-меню.

  1. При раскрытии любого пункта списка открытое содержимое закрывается; по своему действию это похоже на вкладки.
  2. Можно раскрывать любой пункт списка, остальные пункты при этом не закрываются.

Для первого типа аккордеона вставляем <input type="radio">, который позволяет выбрать лишь один пункт, а для второго типа <input type="checkbox"> - с его помощью можно раскрывать и закрывать произвольное число пунктов.

Для каждого <input> необходимо добавить своей уникальный id, а его значение затем вставить в атрибут for у расположенного ниже элемента <label>.

Для того чтобы отдельные пункты были заранее открытими, необходимо добавить к тегу <input> параметр checked.

Архитектура:

<style>
  .spoiler label {cursor:pointer;}
  .spoiler input {display:none;}
  .spoiler div {display:none;}
  .spoiler :checked ~ div {display:block;}
</style>
<div class="spoiler">
  <input type="checkbox" id="spoiler1">
  <label for="spoiler1">Видимый текст</label>
  <div>Скрытый текст</div>
</div>

  <input type="checkbox" id="spoiler3">

</div>

CSS 
.spoiler {border:1px solid #0000fa; margin:5px 0; display:inline-block;}
.spoiler label {cursor:pointer; padding:2px 5px;}
.spoiler :checked + label {border-bottom:1px solid #0000fa;}
.spoiler input, .spoiler div {display: none;}
.st label::before {content:'►'; margin-right:5px;}
.st :checked + label::before {content:'▼';}              /* Открытый спойлер */
.spoiler :checked ~ div {display:block; padding:2px 5px;}/* Открытый спойлер */
.plus label {display:block; width:300px;}
.plus label::after {content: '+'; float: right;} /* Размещаем + по правому краю */
.plus :checked + label::after {content: '−';}
