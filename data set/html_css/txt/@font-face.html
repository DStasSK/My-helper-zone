@font-face

Правило CSS @font-face позволяет определить настройки шрифтов и указать шрифты, которые могут быть загружены либо с удалённого сервера, либо с компьютера пользователя. Это правило может использоваться не только на верхнем уровне CSS, но и внутри любого @-правила.

Правило @font-face не может быть объявлено внутри CSS-селектора.
@font-face {
  имя шрифта;           /* font-family: myFont */
  путь до шрифта;       /* src: url("адрес") */
  параметры;
}
  • имя шрифта - задается как строковый параметр свойства font-family:myFont; этот параметр требуется для того, чтобы правило @font-face было действительным.
  • путь до шрифта - записывается в виде src:url("адрес"), где адрес — относительный или абсолютный путь к файлу. Можно указать несколько значений через запятую. 
    @font-face {
      src: local("имя шрифта"), url("путь_1") format("формат"), url("путь_2");
    }
    • путь - путь к файлу шрифта
    • формат - формат шрифта: woff, woff2, truetype, opentype, embedded-opentype, svg.
Для каждого набора шрифта используются свои настройки @font-face

Если в свойстве src была объявлена функция local(), с названием шрифта, то будет производиться поиск на компьютере пользователя, и в случае обнаружения будет использован этот шрифт. Иначе будет скачан и использован шрифт, указанный в функции url(). Используется первый найденный шрифт - последующие игнорируются.

Веб шрифты попадают под правило ограничения домена (файлы шрифтов должны находиться на том же самом домене, что и использующая их страница), если это ограничение не снимается с помощью CORS.

<style>
   @font-face {
     font-family: myPersonalFont;              /* Имя шрифта */
     src: local("myPersonalFont"),             /* проверка локального шрифта */
          url(./font/myPersonalFont.ttf) format("ttf"),
          url(./font/myPersonalFont.woff) format("woff");
     font-weight: 400;
     font-style: normal;
   }
   P {
     font-family: myPersonalFont, Arial;
   }
</style>

Внутри правила @font-face помимо ссылки на файл шрифта также могут находиться свойства для изменения параметров отображения самого шрифта:

  • font-display
  • font-stretch
  • font-style
  • font-weight
  • font-variant
  • unicode-range
  • ascent-override
  • descent-override

Opera в некоторых случаях может не показывать на веб-странице текст выбранным шрифтом, заменяя его стандартным. Причём для локальных документов всё работает корректно. Это происходит в тех случаях, когда имя пользователя в Windows написано кириллицей.

Браузер Internet Explorer до версии 9 поддерживает только шрифты формата EOT (Embedded OpenType).
@font-face {
  font-family:Righteous;                           /* Имя шрифта */
  src:url(fonts/Righteous/Righteous-Regular.ttf);
}
h1 {
  font-family: 'Righteous', cursive;
}


/* Работающий вариант подключения шрифта для всех версий браузеров */
@font-face {
  font-family: Pompadur;
  src: url(./font/pompadur.eot);                  /* Для IE5+ */
  src: local(pompadur), url(./font/pompadur.ttf); /* Для остальных браузеров */
 }
