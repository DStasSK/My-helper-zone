clip

Данное свойство устарело, вместо него используют clip-path.
развернуть свернуть

Свойство clip определяет область позиционированного элемента, в которой будет показано его содержимое. Всё, что не помещается в эту область, будет обрезано и становится невидимым. На данный момент единственная доступная форма области - прямоугольник.

clip работает только для позиционированных элементов, у которых свойство position задано как absolute или fixed.

Значение по умолчанию - auto, свойство не наследуется, применяется к блочным элементам, анимируется

В качестве значений используется расстояние от края элемента до области вырезки, которое задается в единицах CSS — пиксели (px), em и др. Если край области нужно оставить без изменений, следует установить auto, положение остальных значений показано ниже.

clip: rect(Y1, X2, Y2, X1) | auto

Сначала отсекается верхняя область, потом правая, низ и левая сторона.

Internet Explorer до версии 7 включительно работает с другой формой записи, при которой значения координат разделяются между собой пробелом, а не запятой.

clip: rect(Y1 X2 Y2 X1) | auto

clip-path

Свойство clip-path определяет область позиционированного элемента, в которой будет показано его содержимое. Всё, что не помещается в эту область, будет обрезано и становится невидимым.

clip-path как и clip работает только для позиционированных элементов, у которых свойство position задано как absolute или fixed.

Значение по умолчанию - auto, свойство не наследуется, применяется к блочным элементам, анимируется

clip-path можно использовать совмесно с SVG графикой.

Отсечение с помощью свойства clip-path похоже на вырезание фигуры (например, круга или пятиугольника) из прямоугольного листа бумаги. Свойство относится к спецификации «CSS Masking Module Level 1». Спецификация заявляет: Маскирование CSS предлагает два средства для частичного или полного сокрытия частей визуальных элементов: маскирование и отсечение.

Первая часть, то есть маскирование, связана с использованием графического элемента, такого как изображения PNG, градиента на CSS или элемента SVG, в качестве маски для сокрытия разделов другого элемента.

Вторая часть, то есть clip-path, включает замкнутый векторный контур, который может быть базовой фигурой, определённой в CSS или SVG с использованием тега clipPath. Область внутри этого контура отображается, а всё, что находится за её пределами или снаружи, будет обрезано.

Два способа использования clip-path

1. на CSS

Основные фигуры из «CSS Shapes Module» обеспечивают удобный способ использования clip-path. Доступны разные фигуры: polygon, circle, ellipse и inset; inset предназначен для прямоугольников.

2. на SVG

В качестве альтернативы можно создать фигуру с помощью SVG, а затем обрезать элемент по этой фигуре с помощью синтаксиса URL. Есть два способа сделать это, в обоих случаях в SVG используется элемент clipPath для обёртывания элемента, который определяет обтравочный контур, будь то круг, многоугольник или другой элемент:
а) через указатель на встроенный SVG (разметка SVG существует в самой странице);
б) со ссылкой на внешний документ SVG.

Mожно создать адаптивный обтравочный контур SVG следующим образом:
  • Установить ширину и высоту SVG как 0.
  • Установить идентификатор для элемента clipPath внутри SVG, на который затем можно ссылаться из CSS. Вы можете использовать встроенный или внешний SVG, помня про "поддержку" браузерами.
  • Повторно использовать процентные значения координат многоугольника, определённого с помощью clipPath. Просто разделить их на 100 и добавить как безразмерные точки многоугольника в SVG. Установить значение атрибута clipPathUnits как objectBoundingBox, чтобы обтравочный контур помечал границы элемента HTML, который на него ссылается.
Современные браузеры WebKit и Blink поддерживают обтравочные контуры на SVG, только если они объявлены встроенными (то есть внутри документа). Ссылки на внешние SVG поддерживаются только в Firefox.
Все границы, тени и контуры за пределами области отсечения будут обрезаны. Также сообщается о нескольких ошибках с clip-path и трёхмерными трансформациями, переходами и прозрачностью.

Cинтаксис clip-path

Круг
.cir{clip-path: circle(40% at 20% 50%);}
Эллипс
.ell{clip-path: ellipse(50% 25% at 50% 50%);}
Прямоугольник
/* отступы: сверху справа снизу слева */
.ins{clip-path: inset(10% 20% 0 30%);}
Многоугольник
.pol{clip-path:polygon(
    0 0, 90% 10%, 100% 80%, 20% 100%, 10% 40%
  );}

Синтаксис для базового многоугольника таков:

.element {clip-path: polygon(x1 y1, x2 y2, x3 y3, …);}

Каждая пара аргументов в списке соответствует координатам по оси x и по оси y конкретной вершины многоугольника. Начальную и конечную точку можно не замыкать.

Вот как написано это в примере:

.element {clip-path: polygon(0 0, 90% 10%, 100% 80%, 20% 100%, 10% 40%);}

поддержкa Firefox с указателем на встроенный SVG:

.element {clip-path: url("#clip-shape");}

Tак будет выглядеть селектор с кроссбраузерной поддержкой:

.element {
  -webkit-clip-path: polygon(0 0, 90% 10%, 100% 80%, 20% 100%, 10% 40%);
  clip-path: polygon(0 0, 90% 10%, 100% 80%, 20% 100%, 10% 40%);

  /* требуется для браузеров Webkit/Blink,
     если использовать только встроенный SVG, а не clip-path */
  -webkit-clip-path: url("#clip-shape");
  clip-path: url("#clip-shape");
}

Ниже приведён код встроенного SVG, который нужно будет вставить в любом месте разметки.

<svg width="0" height="0">
  <defs>
    <clipPath id="clip-shape" clipPathUnits="objectBoundingBox">
      <polygon points="0 0, .9 .1, 1 .8, .2 1, .1 .4" />
    </clipPath>
  </defs>
</svg>

"Анимация" базовых фигур через transition, особенности.

При использавании свойства transition число точек, определяющих конечную фигуру, должно быть таким же, как число точек, определяющих начальную фигуру, иначе свойство transition работать не будет, т.е. переход будет мгновенным.

Поскольку шестиугольник содержит шесть пар координатных точек, нужно добавим две пары через дублирование, чтобы их стало восемь и они соответствовали числу пар для восьмиугольника. Эти дублированные пары не будут влиять на форму шестиугольника.

/* шестиугольник с шестью парами координатных точек: */
clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%);

/* первые две пары координат были дублированы: */
clip-path: polygon(50% 0%, 50% 0%, 100% 25%, 100% 25%, 100% 75%, 50% 100%,
                   0% 75%, 0% 25%);

Для браузеров, которые поддерживают обтравочные контуры только на SVG, нужно добавить анимацию SMIL, чтобы получить бесшовный переход при наведении. Согласно спецификации SMIL, декларативные анимации могут применяться для анимации линий и точек многоугольника в SVG, что в настоящее время невозможно с помощью CSS.

1 - SVG
2 - not SVG
