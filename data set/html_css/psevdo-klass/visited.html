Псевдокласс :visited

Псевдокласс :visited устанавливает стиль ссылки по которой был осуществлён переход. При использовании данного параметра, происходит замена стилей в соответствии с выбранными параметрами.

Псевдоэлемент :visited признан небезопасным псевдоэлементом и потому может быть кастомизирован очень ограниченным количеством свойств: color, background-color, border-*-color, outline-color и параметр цвета атрибутов fill и stroke у SVG изображений. Кроме того, для перечисленного списка свойств нельзя установить цвета через rgba() или hsla(); или задать им значение transparent.
<a href="#">по которой был осуществлён переход</a>

a{color: #009}
a:link{color: #963}
a:visited{color: #a0d}
a:active{color: #f00}

a:link:after{content:' (перехода не было)'}
a:visited:after{content:'(переход был)'}     /* этот параметр работать не будет */
a:active:after {content:' (ссылка нажата)'}
Из примера видно, что после перехода по ссылке параметр
a:link:after {content:' (перехода не было)'}
не сбросился, хотя цвет поменялся. Это происходит по причинам безопасности.

Псевдоклассы для оформления стилей могут переопределять друг друга, чтобы избежать этого необходимо их указывать в определенном порядке, а именно:

a:link, a:visited, a:hover, a:active, a:focus {}
