SVG введение

  • Способы вставки изображений в формате SVG на страницу.
  • Размеры изображений в формате SVG.

SVG это язык XML разметки, схожий с XHTML, который может использоваться для рисования векторной графики. Он может быть использован для создания изображения путём указания всех необходимых линий и форм, путём модификации уже существующих растровых изображений, или путём комбинации обоих. Изображения и их компоненты также могут быть трансформированы, собраны вместе, или отфильтрованы чтобы полностью изменить их внешний вид.

Особенности файлов SVG.

Первая важная вещь, которую следует заметить - это порядок отображения элементов. Элементы, которые отрендерились позднее, отображаются поверх предыдущих элементов. Чем дальше вниз элемент (по коду / порядку рендеринга), тем более видимым он будет.

Нормальные файлы SVG - это простые текстовые файлы, содержащие разметку SVG. Рекомендуется расширение ".svg" (все нижним регистром) к имени файла для этих файлов.

<svg width="200" height="140"
     xmlns="http://www.w3.org/2000/svg">

  <rect width="100%" height="100%" fill="#3333fe" />

  <circle cx="100" cy="70" r="50" fill="green" />

  <text x="100" y="82" font-size="40"
        text-anchor="middle" fill="white">SVG</text>
</svg>

Способы вставки изображений в формате SVG на страницу.

Файлы SVG в вебе могут быть отображены прямо в браузере или внедрены в файлы HTML посредством нескольких методов:

  • На файл SVG можно ссылаться с помощью элемента object: 
    <object data="image.svg" type="image/svg+xml" />
  • Аналогично может быть использован элемент iframe: 
    <iframe src="image.svg"></iframe>
  • Аналогично может быть использован элемент embed: 
    <embed src="your.svg" width="199" height="200">
  • Элемент img тоже может быть использован (не работает в Firefox до v 4.0). При такой интеграции SVG на html странице, его содержимое недоступно для JS. 
    <img src="image.svg">
  • SVG изображение может отображаться на html странице как фон элемента, установленный через CSS (не работает в старых браузерах): 
    .el{
       background-image: url(your.png); /* PNG для IE6-8 */
       background-image: url(your.svg);
    }
  • Если HTML является XHTML и выводится как тип приложение/xhtml+xml, SVG может быть прямо внедрён в источник XML.
  • SVG может быть создан динамически с помощью JavaScript и введён в HTML DOM.
Начальное значение свойства display для всех элементов SVG - inline.

Размеры изображений в формате SVG.

Содержимое SVG-элемента отрисовывается на бесконечном холсте и может быть сколь угодно большого размера, но видимая часть холста соответствует размерам SVG-элемента. Эта область отрисовки называется viewport (вьюпорт).

Если просто вставить SVG на страницу и не задавать ему никакие размеры, он отобразится размером 300px на 150px, что не поместилось — обрежется.

Шириной и высотой элемента можно управлять свойствами width и height, эти свойства можно определять как внутри SVG элемента, так и через CSS. Для размеров в пикселях, задаваемых в атрибутах, единицы измерения можно не указывать.


<svg width="250" height="150">


   <circle cx="100" cy="100" r="80" fill="#817" />


</svg>

SVG и тег image

SVG тег image позволяет выводить растровые изображения внутри SVG-объектов.

Спецификация запрашивает приложения для поддержки файлов формата PNG, JPEG и SVG.
<svg width="50" height="50" version="1.1"
     xmlns="http://www.w3.org/2000/svg"
     xmlns:xlink= "http://www.w3.org/1999/xlink">
  <image xlink:href="firefox.jpg" x="0" y="0" height="50px" width="50px"/>
</svg>
  • Если параметры x или y (координаты верхнего левого угла изображения) не заданы, то им будут присвоены значения равные 0.
  • Если параметры height или width не заданы, то им будут присвоены значения равные 0.
  • Если значение любого из параметров изображения height или width равны 0, то изображение будет отключено.

Теги image и img практически синонимы, в процессе парсинга тег image заменяется на img всегда и во всех браузерах. Именно это свойство тега image позволяет использовать его для SVG картинок в браузерах, не поддерживающих SVG. Идея предельно простая: подготовим SVG-файл для хороших браузеров и обычную растровую картинку для остальных. И напишем в HTML такой код:

<svg width="96" height="96">
   <image xlink:href="svg.svg" src="svg.png" width="96" height="96"/>
</svg>

Браузеры, поддерживающие SVG, проигнорировав атрибут src - покажут красивую векторную картинку, прочитав этот код как:

<svg width="96" height="96">
   <image xlink:href="svg.svg" width="96" height="96"/>
</svg>

Браузеры, не поддерживающие SVG, проигнорируют неизвестный им тег svg и, благодаря тому что image и img — это синонимы, прочитают код как:

<img src="svg.png" width="96" height="96"/>
