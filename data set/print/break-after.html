CSS свойство break-after

Добавляет разрыв страницы при печати документа после заданного элемента. Устанавливает, как разрывы страниц, столбцов или регионов должны вести себя после сгенерированного блока. Если сгенерированного блока нет, свойство игнорируется. Значение по умолчанию auto, не наследуется, применяется к блочным элементам.

.elem {
  /* Generic break values */
  break-after: auto | avoid | always | all;

  /* Page break values */
  break-after: avoid-page | page | left | right | recto | verso;

  /* Column break values */
  break-after: avoid-column | column;

  /* Region break values */
  break-after: avoid-region | region;
}
autoВставляет разрыв страницы при необходимости сразу после основного блока.
avoidЗапрещает разрыв страницы после элемента.
alwaysВсегда добавляет разрыв страницы после элемента. Вызывает разрыв страницы сразу после основного блока. Тип этого разрыва — это тип непосредственно содержащегося контекста фрагментации. Если мы находимся внутри мультиколоночного контейнера, то это вызовет разрыв столбца, внутри постраничного носителя (но не внутри контейнера) — разрыв страницы.
Поддерживается только в FF
allВызывает разрыв страницы сразу после основного блока. Прорывая все возможные контексты фрагментации. Таким образом, разрыв внутри мультиколоночного контейнера вызовет разрыв столбца и страницы.

Значения разрыва страницы
avoid-pageПозволяет избежать разрыва страницы сразу после основного блока.
Не поддерживается в FF
pageПринудительно вставляет разрыв страницы сразу после основного блока.
leftПропускает одну или две страницы после элемента, чтобы следующая страница при печати была чётной. Эта страница размещена на левой стороне корешка книги или на задней стороне страницы при двусторонней печати.
rightПропускает одну или две страницы после элемента, чтобы следующая страница при печати была нечётной. Эта страница размещена на правой стороне корешка книги или на передней стороне страницы при двусторонней печати.
rectoПринудительно вставляет один или два разрыва страницы сразу после основного блока, в зависимости от того, какой из них превратит следующую страницу в страницу с обратным направлением. (Страница с обратным направлением — это правая страница в развороте с письмом слева направо или левая страница в развороте с письмом справа налево.)
Не поддерживается в FF
versoПринудительно вставляет один или два разрыва страницы сразу после основного блока, в зависимости от того, какой из них превратит следующую страницу в оборотную страницу. (Оборотная страница — это левая страница на развороте слева направо или правая страница на развороте справа налево.)
Не поддерживается в FF

Значения разрыва столбца
avoid-columnПозволяет избежать разрыва столбца сразу после основного блока.
Не поддерживается в FF
columnПринудительно разрывает столбец сразу после основного блока.
Не поддерживается в FF
Значения разрыва региона
avoid-regionПозволяет избежать разрыва региона сразу после основного блока.
regionПринудительно разрывает область сразу после основного блока.

На каждую возможную точку разрыва (другими словами, на каждую границу элемента) влияют три свойства: значение break-after предыдущего элемента, значение break-before следующего элемента и значение break-inside содержащего элемента. Чтобы определить, нужно ли делать разрыв, применяются следующие правила:

  • Если любое из трех рассматриваемых значений является значением принудительного разрыва (always, left, right, page, column или region), оно имеет приоритет. Если более одного из них являются таким разрывом, берется тот элемент, который появляется последним в потоке (т. е. значение break-before имеет приоритет над значением break-after, которое само по себе имеет приоритет над значением break-inside).
  • Если любое из трех рассматриваемых значений является значением избегания разрыва (avoid, Avoid-page, Avoid-region или Avoid-column), такой разрыв не будет применен в этой точке.

После применения принудительных разрывов при необходимости можно добавлять мягкие разрывы, но не на границах элементов, которые разрешаются в соответствующее избегаемое значение.

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>break-after</title>
    <style>
      @media print {   /* медиазапросы не поддерживется в IE8 */
        .more {break-after: avoid;}
      }
    </style>
  </head>
  <body>
    <h2>Мусорные пакеты</h2>
    <p>...</p>
    <p class="more">Читать дальше</p>
  </body>
</html>
